# Governance

Actioneer is built with governance controls that give your organisation visibility and control over who accesses data, what queries are run, and how AI-generated answers are produced. Nothing is a black box.

---

## Role-Based Access Control

Actioneer enforces role-based access control (RBAC) at the workspace level. Each user is assigned a role that determines what they can see and do.

| Capability | Admin | Analyst | Viewer |
| :--- | :--- | :--- | :--- |
| Connect and manage data sources | Yes | No | No |
| Define and edit metric definitions | Yes | Yes | No |
| Ask questions in Chat | Yes | Yes | Yes |
| Create and edit Boards | Yes | Yes | No |
| Create and push Segments | Yes | Yes | No |
| View Boards and results | Yes | Yes | Yes |
| Manage workspace settings | Yes | No | No |

<Info>
  Roles are managed at the workspace level. A user's role applies across all connected data sources within that workspace.
</Info>

## Audit Logging

Every action in Actioneer is logged. Audit logs provide a full trail of who did what, when, and on which data.

| Log Type | What is captured |
| :--- | :--- |
| **Query logs** | Every question asked, the SQL generated, the data source queried, and the execution time |
| **Access logs** | User logins, role changes, and data source connection events |
| **Change logs** | Updates to metric definitions, Playbook edits, and Segment modifications |
| **Push logs** | Segment pushes to CLM tools — which Segment, which destination, and when |

Logs are retained according to your workspace's configured retention policy and are available for export.

## Explainable AI

Actioneer does not produce opaque answers. Every response is traceable back to the data and logic that produced it.

| Principle | How it works |
| :--- | :--- |
| **SQL visibility** | Every answer in Chat includes the SQL query that generated it. Users can inspect, copy, and validate the query independently. |
| **Source attribution** | Results reference the specific tables, columns, and metric definitions used. |
| **Confidence flagging** | When Actioneer has low confidence in an answer — due to ambiguous questions, missing data, or schema gaps — it flags the result rather than guessing. |
| **No hallucinations by design** | Answers are grounded in your metric definitions and schema, not the model's assumptions. The Memory Layer ensures that business terms are interpreted correctly. |
