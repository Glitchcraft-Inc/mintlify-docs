# Governance

Every question, every answer, every action — **logged and traceable.**

Actioneer is built with governance controls that give your organisation visibility and control over who accesses data, what queries are run, and how AI-generated answers are produced. Nothing is a black box.

---

## Role-Based Access Control

Actioneer enforces role-based access control (RBAC) at the workspace level. Each user is assigned a role that determines what they can see and do.

| &nbsp; Capability | &nbsp; Admin | &nbsp; Analyst | &nbsp; Viewer |
| :--- | :--- | :--- | :--- |
| &nbsp; Connect and manage data sources | &nbsp; Yes | &nbsp; No | &nbsp; No |
| &nbsp; Define and edit metric definitions | &nbsp; Yes | &nbsp; Yes | &nbsp; No |
| &nbsp; Ask questions via Conversational Analytics | &nbsp; Yes | &nbsp; Yes | &nbsp; Yes |
| &nbsp; Create and edit Boards | &nbsp; Yes | &nbsp; Yes | &nbsp; No |
| &nbsp; Create and push Segments | &nbsp; Yes | &nbsp; Yes | &nbsp; No |
| &nbsp; View Boards and results | &nbsp; Yes | &nbsp; Yes | &nbsp; Yes |
| &nbsp; Manage workspace settings | &nbsp; Yes | &nbsp; No | &nbsp; No |

<Info>
  Roles are managed at the workspace level. A user's role applies across all connected data sources within that workspace.
</Info>

## Audit Logging

Every action in Actioneer is logged. Audit logs provide a full trail of who did what, when, and on which data.

| &nbsp; Log Type | &nbsp; What is captured |
| :--- | :--- |
| &nbsp; **Query logs** | &nbsp; Every question asked, the SQL generated, the data source queried, and the execution time |
| &nbsp; **Access logs** | &nbsp; User logins, role changes, and data source connection events |
| &nbsp; **Change logs** | &nbsp; Updates to metric definitions, Playbook edits, and Segment modifications |
| &nbsp; **Push logs** | &nbsp; Segment pushes to CLM tools — which Segment, which destination, and when |

Logs are retained according to your workspace's configured retention policy and are available for export.

## Explainable AI

Actioneer does not produce opaque answers. Every response is traceable back to the data and logic that produced it.

| &nbsp; Principle | &nbsp; How it works |
| :--- | :--- |
| &nbsp; **SQL visibility** | &nbsp; Every answer includes the SQL query that generated it. Users can inspect, copy, and validate the query independently. |
| &nbsp; **Source attribution** | &nbsp; Results reference the specific tables, columns, and metric definitions used. |
| &nbsp; **Confidence flagging** | &nbsp; When Actioneer has low confidence in an answer — due to ambiguous questions, missing data, or schema gaps — it flags the result rather than guessing. |
| &nbsp; **No hallucinations by design** | &nbsp; Answers are grounded in your metric definitions and schema, not the model's assumptions. The Context Engine ensures that business terms are interpreted correctly. |
