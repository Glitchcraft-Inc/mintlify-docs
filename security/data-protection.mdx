# Data Protection

Actioneer is designed to handle sensitive business data without compromising security. All data is encrypted in transit and at rest, credentials are never stored in plain text, and no raw customer data is retained beyond what is needed to serve a query.

---

## Encryption

| Layer | Standard | Details |
| :--- | :--- | :--- |
| **In transit** | TLS 1.3 | All traffic between your infrastructure and Actioneer is encrypted using TLS 1.3 — the latest transport layer security protocol. |
| **At rest** | AES-256 | All stored data, including credentials, configuration, and cached results, is encrypted with AES-256 bit encryption. |

---

## Credential Handling

Actioneer connects to your data warehouse, BI tools, and CLM platforms using credentials you provide — service account keys, API tokens, or access key pairs.

| Practice | Description |
| :--- | :--- |
| **Encrypted storage** | Credentials are encrypted with AES-256 at rest. They are never stored in plain text or logged. |
| **Scoped access** | Each integration uses the minimum permissions required. Data warehouse connections are read-only by default. |
| **No credential sharing** | Credentials are isolated per workspace. They are never shared across organisations or used for any purpose other than the configured integration. |
| **Revocable at any time** | You can disconnect a data source and revoke Actioneer's access from your own platform at any time. |

## Private Cloud Deployment

For organisations with strict data sovereignty or network isolation requirements, Actioneer offers custom private cloud deployment. Your instance runs in a dedicated environment — no shared infrastructure with other tenants.

---

## Read-Only by Default

Actioneer operates as a read-only integration. It queries your data but does not create, modify, or delete anything in your warehouse, BI tool, or CLM platform.

| What Actioneer can do | What Actioneer cannot do |
| :--- | :--- |
| Run `SELECT` queries on your warehouse | Insert, update, or delete data |
| Read datasets and reports from BI tools | Create, edit, or publish reports |
| Query event data and user profiles | Modify events, properties, or schemas |
| Push audience segments to CLM tools (when configured) | Send messages directly to end users |

## Third-Party Security Audits

Actioneer undergoes regular third-party security audits to validate controls and identify vulnerabilities. Audit findings are remediated on a defined timeline and tracked to closure.

| Audit Type | Frequency |
| :--- | :--- |
| **SOC 2 Type II** | Annual |
| **ISO 27001 surveillance** | Annual |
| **Penetration testing** | Regular, conducted by independent third parties |
