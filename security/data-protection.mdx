# Data Protection

Your data never leaves your infrastructure. **We prove it.**

Actioneer is designed to handle sensitive business data without compromising security. All data is encrypted in transit and at rest, credentials are never stored in plain text, and no raw customer data is retained beyond what is needed to serve a query.

---

## Deployment Model

Actioneer does not ingest, copy, or relocate your data. It connects to your existing data sources and queries them in place.

| &nbsp; Principle | &nbsp; Description |
| :--- | :--- |
| &nbsp; **In-place querying** | &nbsp; Actioneer reads data where it lives — in your warehouse, BI tool, or CRM. Nothing is moved. |
| &nbsp; **No data pipeline** | &nbsp; No ETL required. No staging tables. No data duplication. |
| &nbsp; **Read-only by default** | &nbsp; Actioneer runs `SELECT` queries. It cannot insert, update, or delete data in your connected sources. |
| &nbsp; **Your infrastructure, your control** | &nbsp; Credentials are yours. You can disconnect a data source and revoke access from your own platform at any time. |

## Encryption

### AES-256 at rest

All stored data — credentials, configuration, cached results — is encrypted using **AES-256**. This is the same encryption standard used by banks, governments, and military systems worldwide. "256" refers to the key length (256 bits), which means there are more possible key combinations than atoms in the observable universe. Even with the fastest computers available today, brute-forcing this encryption would take longer than the age of the universe.

In practical terms: if someone gained physical access to Actioneer's storage, the data would be unreadable without the encryption key.

### TLS 1.3 in transit

All data moving between your systems and Actioneer is encrypted using **TLS 1.3** — the latest version of the Transport Layer Security protocol. TLS creates an encrypted tunnel between two points so that data cannot be intercepted or read while in transit. Version 1.3 is faster and more secure than its predecessors, removing outdated cryptographic algorithms and reducing the handshake to a single round trip.

In practical terms: everything flowing over the network — queries, results, credentials — is encrypted from the moment it leaves your infrastructure until it arrives at Actioneer, and vice versa.

## Source Read Access

Actioneer connects to your data warehouse, BI tools, and CLM platforms using credentials you provide — service account keys, API tokens, or access key pairs.

| &nbsp; Practice | &nbsp; Description |
| :--- | :--- |
| &nbsp; **Encrypted storage** | &nbsp; Credentials are encrypted with AES-256 at rest. They are never stored in plain text or logged. |
| &nbsp; **Scoped access** | &nbsp; Each integration uses the minimum permissions required. Data warehouse connections are read-only by default. |
| &nbsp; **No credential sharing** | &nbsp; Credentials are isolated per workspace. They are never shared across organisations or used for any purpose other than the configured integration. |
| &nbsp; **Revocable at any time** | &nbsp; You can disconnect a data source and revoke Actioneer's access from your own platform at any time. |

## Read-Only Operations

Actioneer operates as a read-only integration. It queries your data but does not create, modify, or delete anything in your warehouse, BI tool, or CLM platform.

| &nbsp; What Actioneer can do | &nbsp; What Actioneer cannot do |
| :--- | :--- |
| &nbsp; Run `SELECT` queries on your warehouse | &nbsp; Insert, update, or delete data |
| &nbsp; Read datasets and reports from BI tools | &nbsp; Create, edit, or publish reports |
| &nbsp; Query event data and user profiles | &nbsp; Modify events, properties, or schemas |
| &nbsp; Push audience segments to CLM tools (when configured) | &nbsp; Send messages directly to end users |
