# Responsible AI Practices

Actioneer is built on the principle that AI should be trustworthy, explainable, and accountable. Every answer, every agent decision, and every automated workflow is governed by the same standards.

---

## Fairness

Actioneer's outputs are impartial, neutral, and fact-based.

| Principle | How it works |
| :--- | :--- |
| **Grounded in data** | Every answer is derived from your governed data and metric definitions — not model opinions or assumptions |
| **No editorial bias** | Results reflect what the data shows, not what the model thinks you want to hear |
| **Consistent definitions** | The Context Engine ensures the same question always uses the same business logic, regardless of who asks |
| **Neutral anomaly detection** | Scouts flag deviations from expected ranges — they do not interpret or editorialize |

---

## Transparency

Every AI-generated output in Actioneer is traceable and auditable.

| Principle | How it works |
| :--- | :--- |
| **SQL visibility** | Every answer includes the SQL query that produced it — visible, copyable, and independently verifiable |
| **Reasoning shown** | The logic path from question to answer is displayed alongside the result |
| **Confidence flagging** | When Actioneer has low confidence in an answer — due to ambiguous questions, missing data, or schema gaps — it flags the result rather than guessing |
| **No hallucinations by design** | Answers are grounded in the Context Engine, not the model's training data. If the data doesn't support an answer, Actioneer says so. |
| **Audit trail** | Every query, agent decision, and Playbook run is logged with timestamp, user, and execution details |

---

## Privacy

Actioneer respects the boundaries of your data at every layer.

| Principle | How it works |
| :--- | :--- |
| **Read-only by default** | Actioneer queries your data in place. It does not create, modify, or delete anything in your warehouse or connected tools. |
| **In-place querying** | Data is never copied, ingested, or relocated. Actioneer reads it where it lives. |
| **No raw data retention** | Query results are not stored beyond the active session unless explicitly saved to a Board |
| **Credential isolation** | Each workspace's credentials are isolated. No cross-tenant data access. |
| **Data minimisation** | Each query only reads the tables and columns relevant to the question |

---

## Security

Enterprise-grade security is built into the platform, not added as an afterthought.

| Certification | Status |
| :--- | :--- |
| **SOC 2 Type II** | Certified — annual audit covering security, availability, and confidentiality |
| **ISO 27001** | Certified — audited by Schellman |
| **GDPR** | Compliant — data minimisation, subject rights, 72-hour breach notification |

| Control | Description |
| :--- | :--- |
| **Encryption at rest** | AES-256 — all stored data is encrypted using the same standard banks and governments use |
| **Encryption in transit** | TLS 1.3 — all data moving between your systems and Actioneer is encrypted in the latest transport protocol |
| **RBAC** | Role-based access control — who builds what, who runs what, who sees what |
| **Private cloud deployment** | Available for institutions that require dedicated infrastructure |
| **Data residency** | US, EU, and APAC options |
