# Responsible AI Practices

No black boxes. **Every AI decision is explainable, auditable, and governed.**

Actioneer is built on the principle that AI should be trustworthy, explainable, and accountable. Every answer, every agent decision, and every automated workflow is governed by the same standards.

---

## Fairness

Actioneer's outputs are impartial, neutral, and fact-based.

| &nbsp; Principle | &nbsp; How it works |
| :--- | :--- |
| &nbsp; **Grounded in data** | &nbsp; Every answer is derived from your governed data and metric definitions — not model opinions or assumptions |
| &nbsp; **No editorial bias** | &nbsp; Results reflect what the data shows, not what the model thinks you want to hear |
| &nbsp; **Consistent definitions** | &nbsp; The Context Engine ensures the same question always uses the same business logic, regardless of who asks |
| &nbsp; **Neutral anomaly detection** | &nbsp; Scouts flag deviations from expected ranges — they do not interpret or editorialize |

---

## Transparency

Every AI-generated output in Actioneer is traceable and auditable.

| &nbsp; Principle | &nbsp; How it works |
| :--- | :--- |
| &nbsp; **SQL visibility** | &nbsp; Every answer includes the SQL query that produced it — visible, copyable, and independently verifiable |
| &nbsp; **Reasoning shown** | &nbsp; The logic path from question to answer is displayed alongside the result |
| &nbsp; **Confidence flagging** | &nbsp; When Actioneer has low confidence in an answer — due to ambiguous questions, missing data, or schema gaps — it flags the result rather than guessing |
| &nbsp; **No hallucinations by design** | &nbsp; Answers are grounded in the Context Engine, not the model's training data. If the data doesn't support an answer, Actioneer says so. |
| &nbsp; **Audit trail** | &nbsp; Every query, agent decision, and Playbook run is logged with timestamp, user, and execution details |

## Privacy

Actioneer is designed around data minimisation and boundary respect. The platform queries your data in place, never copies or relocates it, and retains no raw results beyond the active session. Full details on encryption, credential handling, and data residency are covered in Data Protection.
