Send one envelope per request. The same schema, authentication, and idempotency rules apply as `/batch` — reuse a stable `event_id` so retries collapse to a single event.

<Info>
  `event_id` is the deduplication key: reuse it across retries of the same logical event so they collapse to one. See [Reliability](/ingestion/reliability) and [Errors & Troubleshooting](/ingestion/errors).
</Info>
