Forward many envelopes at once for grouped sends and high-throughput pipelines. A single `X-API-Key` authenticates every envelope; attribute each one explicitly via `distinct_id` and order via `timestamp`.

<Info>
  Each envelope carries its own `event_id` for deduplication — reuse it across retries so the batch can't double-count. See [Reliability](/ingestion/reliability) and [Errors & Troubleshooting](/ingestion/errors).
</Info>
