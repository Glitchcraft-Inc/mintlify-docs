Forward Firebase / Google Analytics 4 events to Actioneer. Unlike the webhook-native SDKs, Firebase has **no direct outbound webhook** — you forward via a Cloud Function or the BigQuery export.

<Info>
  Scaffold recipe — Firebase requires a small forwarding component. Pick the path that matches your latency needs and confirm against your Firebase project.
</Info>

## How it works

Two common paths:

- **Real-time** — a **Cloud Function** triggered on analytics conversion events that builds an Actioneer envelope and POSTs to `/capture`.
- **Batch** — the **BigQuery export** of GA4 events, read on a schedule and forwarded to `/batch` (higher latency, complete coverage).

## Steps

<Steps>
  <Step title="Choose a forwarding path">
    Real-time Cloud Function for low latency, or scheduled BigQuery-export forwarding for completeness.
  </Step>
  <Step title="Build the envelope">
    Map GA4 fields into the Actioneer envelope per the mapping below. Convert `event_timestamp` (microseconds) to ISO-8601 UTC.
  </Step>
  <Step title="Send and confirm">
    POST to `https://api.actioneer.com/capture` with `X-API-Key: <api-key>` and confirm `{ "status": 1 }`.
  </Step>
</Steps>

## What gets forwarded

| Firebase / GA4 | Actioneer |
| :--- | :--- |
| event `name` | `event` |
| event `params` | `properties` |
| `user_properties` | `properties.$set` |
| `user_id` | `distinct_id` (post-login) |
| `app_instance_id` / pseudo ID | `distinct_id` (pre-login); `$anon_distinct_id` on `$identify` |
| generated UUID (per event) | `event_id` (dedup) |
| `event_timestamp` (µs) → ISO-8601 | `timestamp` |

```js
// Cloud Function — illustrative
exports.forwardToActioneer = functions.analytics.event("purchase").onLog(async (e) => {
  const ts = new Date(Number(e.reportingDate)).toISOString();
  await fetch("https://api.actioneer.com/capture", {
    method: "POST",
    headers: { "Content-Type": "application/json", "X-API-Key": process.env.ACTIONEER_KEY },
    body: JSON.stringify({
      event_id: e.params.event_id,
      event: e.name,
      distinct_id: e.user?.userId || e.user?.appInstanceId,
      timestamp: ts,
      properties: e.params
    })
  });
});
```

<Warning>
  Map `app_instance_id` (or `user_pseudo_id`) as the pre-login `distinct_id`, and emit `$identify` with `$anon_distinct_id` when `user_id` becomes known. See [Identity Model](/ingestion/identity).
</Warning>
