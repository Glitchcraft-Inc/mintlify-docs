Send analytics events, identity calls, and profile updates to Actioneer **directly from any event-generation SDK — no warehouse round-trip.**

Where data-source connectors let Actioneer *read* from your stack, the Ingestion API lets your app *push* events in real time. One JSON-over-HTTPS envelope carries event properties, profile updates, and session association together. If you already emit analytics through Firebase, Jitsu, RudderStack, or Segment, you forward those events straight to Actioneer.

| At a glance | |
| :--- | :--- |
| **Transport** | HTTPS / JSON |
| **Base URL** | `https://api.actioneer.com` |
| **Audience** | Client / data engineering |
| **Auth** | `X-API-Key` header (server-side secret) |

## Before you build

<Warning>
  The API key is a server-side secret. Issue requests from a backend or trusted server context — do not embed it in distributable client binaries or web bundles. Never place secrets, credentials, tokens, or cardholder data inside event or profile properties.
</Warning>

<Info>
  The contract is intentionally analytics-shaped (Segment-style events, identify calls, user properties), so wiring up an existing SDK is mostly a change of transport and identity handling rather than a re-modeling of your events.
</Info>
