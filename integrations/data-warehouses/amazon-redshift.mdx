# Linking Redshift as a Source

This guide walks through connecting a Redshift cluster to Actioneer. The setup creates a dedicated database user with `SELECT`-only permissions on the schemas you specify.

| | |
| :--- | :--- |
| &nbsp; **Prerequisites** | &nbsp; AWS account with a Redshift cluster, Actioneer account |
| &nbsp; **Access granted** | &nbsp; `SELECT` queries on specified schemas |
| &nbsp; **Estimated time** | &nbsp; ~20 minutes |
| &nbsp; **Actioneer IP** | &nbsp; `35.244.14.238` |

<Info>
  Actioneer cannot insert, update, delete, or modify any data, tables, or cluster settings.
</Info>

---

<Steps>
  <Step title="Locate your Redshift cluster endpoint and database name">
    Two values are needed from the AWS Console.

    1. Sign in at [console.aws.amazon.com](https://console.aws.amazon.com) and open the **Amazon Redshift** service.
    2. Click **Clusters** in the left sidebar and select your cluster.
    3. On the **General information** tab, copy the **Endpoint**. It follows the format `my-cluster.abc123.us-east-1.redshift.amazonaws.com:5439`. The address before the colon is the **Host**. The port is always `5439`.
    4. Note the **Database name** shown on the same page (e.g., `analytics`).

    | &nbsp; Field | &nbsp; Location | &nbsp; Value Example |
    | :--- | :--- | :--- |
    | &nbsp; **Host** | &nbsp; Cluster endpoint (without port) | &nbsp; `my-cluster.abc123.us-east-1.redshift.amazonaws.com` |
    | &nbsp; **Port** | &nbsp; Always `5439` for Redshift | &nbsp; `5439` |
    | &nbsp; **Database** | &nbsp; General information tab | &nbsp; `analytics` |
  </Step>
  <Step title="Create a read-only database user">
    Connect to your Redshift cluster using a SQL client (Redshift Query Editor, DBeaver, or psql) and run the following. Replace `YOUR_SCHEMA` with the schema Actioneer should query.

    ```sql expandable
    -- Create a dedicated read-only user
    CREATE USER actioneer_readonly
      PASSWORD 'Choose-A-Strong-Password-Here'
      CONNECTION LIMIT 5;
    
    -- Grant schema access
    GRANT USAGE ON SCHEMA YOUR_SCHEMA TO actioneer_readonly;
    
    -- Grant read access to all existing tables
    GRANT SELECT ON ALL TABLES IN SCHEMA YOUR_SCHEMA TO actioneer_readonly;
    
    -- Ensure future tables are also readable
    ALTER DEFAULT PRIVILEGES IN SCHEMA YOUR_SCHEMA
      GRANT SELECT ON TABLES TO actioneer_readonly;
    ```

    <Note>
      Replace `YOUR_SCHEMA` with your actual schema name (e.g., `public` or `reporting`). Run the `GRANT` statements for each schema Actioneer should access.
    </Note>

    <Warning>
      Save the password you set for `actioneer_readonly`. You will enter it in Actioneer in the next step. Store it in a password manager.
    </Warning>
  </Step>
  <Step title="Connect Redshift in Actioneer">
    1. In Actioneer, click **Data** in the left sidebar.
    2. Select **Amazon Redshift** from the list of data sources.
    3. When prompted, select **Direct Connection**.
    4. Complete the connection form:

    | &nbsp; Field | &nbsp; Value |
    | :--- | :--- |
    | &nbsp; **Connection Name** | &nbsp; A descriptive label, e.g. `Production Redshift` |
    | &nbsp; **Host** | &nbsp; From Step 1 — cluster endpoint without the port |
    | &nbsp; **Port** | &nbsp; `5439` |
    | &nbsp; **Database Name** | &nbsp; From Step 1 (e.g., `analytics`) |
    | &nbsp; **Username** | &nbsp; `actioneer_readonly` |
    | &nbsp; **Password** | &nbsp; The password set in Step 2 |

    5. Click **Connect**.
  </Step>
</Steps>

## Common questions

<AccordionGroup>
  <Accordion title="Can Actioneer modify data in Redshift?">
    No. The `actioneer_readonly` user holds only `SELECT` and `USAGE` grants. It cannot insert, update, delete, or drop any objects in your cluster.
  </Accordion>

  <Accordion title="How does Actioneer protect credentials?">
    All credentials are encrypted with AES-256 at rest and transmitted over TLS 1.3 protocol. They are never stored in plain text or logged. Actioneer is SOC 2 Type II certified, ISO 27001 certified (audited by Schellman), and GDPR compliant.
  </Accordion>
</AccordionGroup>
