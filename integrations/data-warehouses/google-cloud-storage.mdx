# Linking Google Cloud Storage as a Source

This guide walks through connecting a Google Cloud Storage (GCS) bucket to Actioneer. The setup uses a GCP service account with read-only access scoped to your specific bucket.

| | |
| :--- | :--- |
| &nbsp; **Prerequisites** | &nbsp; GCP project with GCS bucket, Actioneer account |
| &nbsp; **Access granted** | &nbsp; Read-only object access on specified bucket |
| &nbsp; **Estimated time** | &nbsp; ~15 minutes |

<Info>
  Actioneer cannot upload, overwrite, delete, or modify any files or bucket settings.
</Info>

---

<Steps>
  <Step title="Note your GCS bucket name">
    1. Sign in at [console.cloud.google.com](https://console.cloud.google.com).
    2. In the left sidebar, open **Cloud Storage → Buckets**.
    3. Locate your bucket and copy the exact **Bucket Name**.

    | &nbsp; Field | &nbsp; Where to find it | &nbsp; Example |
    | :--- | :--- | :--- |
    | &nbsp; **Bucket Name** | &nbsp; Cloud Storage → Buckets list | &nbsp; `my-data-bucket` |
  </Step>

  <Step title="Create a service account and grant Storage roles">
    Actioneer authenticates using a GCP service account JSON key. This keeps credentials under your control.

    1. Open [IAM & Admin → Service Accounts](https://console.cloud.google.com/iam-admin/serviceaccounts).
    2. Click **Create Service Account**.
    3. Name it something clear (e.g., `actioneer-gcs-reader`). Click **Create and Continue**.
    4. Assign one of the following roles:

    | &nbsp; Role | &nbsp; Role ID | &nbsp; Purpose |
    | :--- | :--- | :--- |
    | &nbsp; Storage Object Viewer | &nbsp; `roles/storage.objectViewer` | &nbsp; Read-only object access |

    <Tip>
      `Storage Object Viewer` is the least-privilege option and is recommended. Only use `Storage Object Admin` if Actioneer needs to access object metadata beyond standard reads.
    </Tip>

    5. Click **Continue**, then **Done**.

    **Download the JSON key**

    1. On the Service Accounts page, click the account you just created.
    2. Go to the **Keys** tab → **Add Key** → **Create New Key**.
    3. Select **JSON** and click **Create**. The key file downloads automatically.

    <Warning>
      Store this JSON key file securely. It grants read access to your GCS bucket. Do not commit it to version control or share it over unencrypted channels.
    </Warning>
  </Step>

  <Step title="Connect Google Cloud Storage in Actioneer">
    1. In Actioneer, click **Data** in the left sidebar.
    2. Select **Google Cloud Storage** from the list of data sources.
    3. When prompted, select **Direct Connection**.
    4. Complete the connection form:

    | &nbsp; Field | &nbsp; Value |
    | :--- | :--- |
    | &nbsp; **Connection Name** | &nbsp; e.g. `Production GCS Bucket` |
    | &nbsp; **Bucket Name** | &nbsp; From Step 1 |
    | &nbsp; **Service Account JSON Key** | &nbsp; Upload the JSON key from Step 2 |

    5. Click **Connect**.
  </Step>
</Steps>

## Common questions

<AccordionGroup>
  <Accordion title="Can Actioneer modify files in my GCS bucket?">
    No. The Storage Object Viewer role grants read-only access to objects. Actioneer cannot upload, overwrite, delete, or modify any files or bucket settings.
  </Accordion>

  <Accordion title="How does Actioneer protect credentials?">
    All credentials are encrypted with AES-256 at rest and transmitted over TLS 1.3 protocol. They are never stored in plain text or logged. Actioneer is SOC 2 Type II certified, ISO 27001 certified (audited by Schellman), and GDPR compliant.
  </Accordion>
</AccordionGroup>
