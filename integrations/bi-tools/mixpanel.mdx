# Linking Mixpanel as a Source

This guide walks through connecting a Mixpanel project to Actioneer. By the end, Actioneer will be able to query your event data and user profiles.

|  |  |
| :-- | :-- |
| **Prerequisites** | Mixpanel account with Project Admin access, Actioneer account |
| **Access granted** | Event data and user profile queries via the Mixpanel API |
| **Estimated time** | 10 minutes |

<Info>
  Actioneer cannot create, edit, or delete events, user properties, or reports in your Mixpanel project.
</Info>

---

<Steps>
  <Step title="Find your Mixpanel Project ID">
    The Project ID identifies which Mixpanel project Actioneer should connect to.

    1. Sign in at [mixpanel.com](https://mixpanel.com).
    2. Click your project name in the top-left corner and select the target project.
    3. Go to **Settings** (gear icon, top-right) → **Project Settings**.
    4. Copy the **Project ID** shown at the top of the page (a numeric ID, e.g., `12345678`).

    | Field | Location | Value Example |
    | :-- | :-- | :-- |
    | **Project ID** | Project Settings, top of page | `12345678` |
  </Step>
  <Step title="Create a Service Account for Actioneer">
    A Mixpanel Service Account is a machine-to-machine credential, separate from personal logins. It provides scoped access to your project's data.

    1. In Mixpanel, go to **Settings → Project Settings → Service Accounts**.
    2. Click **Add Service Account**.
    3. Name it something clear, e.g. `Actioneer Integration`.
    4. Set the role to **Analyst**. This limits access to querying data — no administrative permissions.
    5. Click **Create**.
    6. Copy the **Username** and **Secret** immediately.

    | Field | Location | Value Example |
    | :-- | :-- | :-- |
    | **Username** | Service Account creation dialogue | `abc123.mp-service-acct` |
    | **Secret** | Service Account creation dialogue — shown once | `wZ4xR9qL2vB7...` |

    <Warning>
      Mixpanel displays the Secret exactly once. If you close the dialogue without copying it, you will need to delete the Service Account and create a new one. Store both values in a password manager.
    </Warning>
  </Step>
  <Step title="Connect Mixpanel in Actioneer">
    1. In Actioneer, click **Data** in the left sidebar.
    2. Select **Mixpanel** from the list of data sources.
    3. Complete the connection form:

    | Field | Value |
    | :-- | :-- |
    | **Connection Name** | A descriptive label, e.g. `Production Mixpanel` |
    | **Project ID** | From Step 1 |
    | **Service Account Username** | From Step 2 |
    | **Service Account Secret** | From Step 2 |
    | **Data Center** | `US` or `EU` — match your Mixpanel project's data residency setting |

    4. Click **Connect**.
  </Step>
</Steps>

## Common questions

<AccordionGroup>
  <Accordion title="What data can Actioneer read from Mixpanel?">
    Actioneer can query event data, user profiles, and reports accessible via the Mixpanel API — the same data visible to the Analyst role. It cannot create, modify, or delete events, user properties, or reports.
  </Accordion>

  <Accordion title="How does Actioneer protect credentials?">
    All credentials are encrypted with AES-256 at rest and transmitted over TLS 1.3 protocol. They are never stored in plain text or logged. Actioneer is SOC 2 Type II certified, ISO 27001 certified (audited by Schellman), and GDPR compliant.
  </Accordion>
</AccordionGroup>
