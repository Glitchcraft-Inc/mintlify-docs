# Linking Amplitude as a Source

This guide walks through connecting an Amplitude project to Actioneer. By the end, Actioneer will be able to query your event data and charts.

| | |
| :--- | :--- |
| &nbsp; **Prerequisites** | &nbsp; Amplitude account with project-level access, Actioneer account |
| &nbsp; **Access granted** | &nbsp; Event data and chart queries via the Amplitude API |
| &nbsp; **Estimated time** | &nbsp; ~10 minutes |

<Info>
  Actioneer cannot create, modify, or delete any events, properties, or configuration in your Amplitude project.
</Info>

## Connect via Fivetran

Actioneer uses [Fivetran](https://fivetran.com) to sync Amplitude data into your warehouse. Fivetran handles extraction and scheduling — Actioneer queries the synced tables.

<Steps>
  <Step title="Start the connection in Actioneer">
    1. In Actioneer, click **Data** in the left sidebar.
    2. Select **Amplitude** from the list of data sources.
    3. Enter a **Connection Name** — a descriptive label to identify this source (e.g., `Production Amplitude`).
    4. Click **Connect**.
    5. Actioneer creates the warehouse connection and displays a **Connection Created** confirmation.
    6. Click **Continue to Amplitude** — a new tab opens to the Fivetran configuration page.
  </Step>
  <Step title="Choose a sync method in Fivetran">
    Fivetran supports two ways to retrieve Amplitude events:

    | | |
    | :--- | :--- |
    | &nbsp; **Amplitude API** | &nbsp; Pulls events directly through the Amplitude Export API. Requires an Amplitude account. |
    | &nbsp; **Cloud Storage Export** | &nbsp; Reads events from a Google Cloud Storage bucket where Amplitude exports data. Requires both an Amplitude account and a GCS destination configured in Amplitude. |

    For most setups, the **API method** is the simplest path. Toggle **Use Cloud Storage** on only if you already have Amplitude's Cloud Storage Export configured.
  </Step>
  <Step title="Find your Amplitude API credentials">
    1. Sign in at [app.amplitude.com](https://app.amplitude.com).
    2. Click your organization name in the top-left and select the target project.
    3. Go to **Settings** (gear icon) → **Projects** → select the project.
    4. Copy the **API Key** and **Secret Key**.

    | &nbsp; Field | &nbsp; Location |
    | :--- | :--- |
    | &nbsp; **API Key** | &nbsp; Project Settings → General |
    | &nbsp; **Secret Key** | &nbsp; Project Settings → General |

    <Warning>
      The Secret Key grants data export access. Treat it like a password — do not share it over unencrypted channels or commit it to version control.
    </Warning>
  </Step>
  <Step title="Add your project in Fivetran">
    Back on the Fivetran configuration page:

    1. Click **+ Add Project**.
    2. Enter the **API Key** and **Secret Key** from Step 3.
    3. Add additional projects if needed — each requires its own key pair.
    4. Click **Save & Test**. Fivetran validates the connection and begins the initial sync.

    <Tip>
      If you use Amplitude's Cloud Storage Export, you will also need to provide your **GCS bucket name** and grant Fivetran read access to the bucket. See [Fivetran's Amplitude setup guide](https://fivetran.com/docs/connectors/applications/amplitude/setup-guide) for Cloud Storage–specific steps.
    </Tip>
  </Step>
</Steps>

## What syncs

Fivetran syncs the following Amplitude data into your warehouse:

| &nbsp; Table | &nbsp; Description |
| :--- | :--- |
| &nbsp; **events** | &nbsp; Raw event data including event type, user ID, device, timestamp, and event properties |
| &nbsp; **merge_ids** | &nbsp; User identity merge mappings |
| &nbsp; **event_types** | &nbsp; Lookup of event type definitions |

Sync frequency and historical backfill depth are configured in Fivetran. Actioneer queries whatever lands in the warehouse.

## Common questions

<AccordionGroup>
  <Accordion title="What Amplitude data can Actioneer access?">
    Actioneer queries the tables Fivetran syncs into your warehouse — event data, user properties, and identity mappings. It cannot access Amplitude directly or modify any data in your Amplitude project.
  </Accordion>

  <Accordion title="How does Actioneer protect credentials?">
    All credentials are encrypted with AES-256 at rest and transmitted over TLS 1.3 protocol. They are never stored in plain text or logged. Actioneer is SOC 2 Type II certified, ISO 27001 certified (audited by Schellman), and GDPR compliant.
  </Accordion>

  <Accordion title="Can I control how far back Fivetran syncs?">
    Yes. Fivetran's Amplitude connector supports configurable historical sync depth. Set this during initial setup or adjust it later in your Fivetran dashboard.
  </Accordion>
</AccordionGroup>
