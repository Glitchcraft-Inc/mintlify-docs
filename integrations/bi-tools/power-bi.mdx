# Linking Power BI as a Source

This guide walks through connecting a Microsoft Power BI workspace to Actioneer. By the end, Actioneer will be able to query your datasets and read report data.

| | |
| :--- | :--- |
| &nbsp; **Prerequisites** | &nbsp; Azure AD account, Power BI Pro or Premium workspace, Actioneer account |
| &nbsp; **Access granted** | &nbsp; Dataset queries and report reads via Fivetran sync |
| &nbsp; **Estimated time** | &nbsp; ~20 minutes |

<Info>
  Actioneer cannot create, edit, publish, or delete any reports, dashboards, or datasets in your Power BI workspace.
</Info>

## Connect via Fivetran

Actioneer uses [Fivetran](https://fivetran.com) to sync Power BI metadata and dataset information into your warehouse. Fivetran handles extraction and scheduling — Actioneer queries the synced tables.

<Steps>
  <Step title="Start the connection in Actioneer">
    1. In Actioneer, click **Data** in the left sidebar.
    2. Select **Microsoft Power BI** from the list of data sources.
    3. Enter a **Connection Name** — a descriptive label to identify this source (e.g., `Company Power BI`).
    4. Click **Connect**.
    5. Actioneer creates the warehouse connection and displays a **Connection Created** confirmation.
    6. Click **Continue to Power BI** — a new tab opens to the Fivetran configuration page.
  </Step>
  <Step title="Register an Azure AD app">
    Fivetran authenticates through Microsoft Entra ID (formerly Azure Active Directory). You register a dedicated app that acts as the connector's identity.

    1. Sign in at [portal.azure.com](https://portal.azure.com).
    2. Search for **App registrations** and click **New registration**.
    3. Name it `Fivetran Power BI`. Leave the other fields as defaults. Click **Register**.
    4. On the app overview page, copy the **Application (client) ID** and the **Directory (tenant) ID**.
    5. In the left sidebar, click **Certificates & secrets** → **New client secret**.
    6. Add a description (e.g., `Fivetran`), choose an expiry period (1 or 2 years), and click **Add**.
    7. Copy the **Value** of the secret immediately.

    | &nbsp; Field | &nbsp; Location |
    | :--- | :--- |
    | &nbsp; **Client ID** | &nbsp; App overview → Application (client) ID |
    | &nbsp; **Tenant ID** | &nbsp; App overview → Directory (tenant) ID |
    | &nbsp; **Client Secret** | &nbsp; Certificates & secrets → Value |

    <Warning>
      Azure displays the client secret Value exactly once. If you leave the page without copying it, you will need to delete the secret and create a new one.
    </Warning>
  </Step>
  <Step title="Grant API permissions">
    The registered app needs permission to read Power BI data.

    1. In the Azure portal, go to your app registration.
    2. Click **API permissions** → **Add a permission**.
    3. Select **Power BI Service** and add the following **Delegated permissions**:
       - `Tenant.Read.All`
    4. Click **Grant admin consent** to activate the permission.

    <Tip>
      If you are a Fabric administrator, you can authenticate directly. Otherwise, configure a service principal — see [Microsoft's service principal documentation](https://learn.microsoft.com/en-us/power-bi/developer/embedded/embed-service-principal) for details.
    </Tip>
  </Step>
  <Step title="Complete the Fivetran configuration">
    On the Fivetran configuration page:

    | &nbsp; Field | &nbsp; Value |
    | :--- | :--- |
    | &nbsp; **Tenant ID** | &nbsp; From Step 2 |
    | &nbsp; **Client ID** | &nbsp; From Step 2 |
    | &nbsp; **Client secret** | &nbsp; From Step 2 |

    Click **Save & Test**. Fivetran validates the connection and begins the initial sync.
  </Step>
</Steps>

## What syncs

Fivetran syncs the following Power BI metadata into your warehouse:

| &nbsp; Table | &nbsp; Description |
| :--- | :--- |
| &nbsp; **dataset** | &nbsp; Dataset metadata including name, configuration, and refresh schedules |
| &nbsp; **report** | &nbsp; Report definitions and workspace associations |
| &nbsp; **dashboard** | &nbsp; Dashboard metadata and tile layouts |
| &nbsp; **workspace** | &nbsp; Workspace details and membership |
| &nbsp; **refresh_history** | &nbsp; Dataset refresh logs and status |

Sync frequency is configured in Fivetran. Actioneer queries whatever lands in the warehouse.

## Common questions

<AccordionGroup>
  <Accordion title="Can Actioneer modify reports or dashboards in Power BI?">
    No. Actioneer reads synced metadata from your warehouse. It cannot create, edit, publish, or delete any reports, dashboards, or datasets in your Power BI workspace.
  </Accordion>

  <Accordion title="How does Actioneer protect credentials?">
    All credentials are encrypted with AES-256 at rest and transmitted over TLS 1.3 protocol. They are never stored in plain text or logged. Actioneer is SOC 2 Type II certified, ISO 27001 certified (audited by Schellman), and GDPR compliant.
  </Accordion>

  <Accordion title="What Power BI license do I need?">
    A Power BI Pro account is required at minimum. Power BI Premium provides additional features. You also need a tenant in Microsoft Entra ID (Azure AD) to register the app.
  </Accordion>
</AccordionGroup>
