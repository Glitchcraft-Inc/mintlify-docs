# Linking Power BI as a Source

This guide walks through connecting a Microsoft Power BI workspace to Actioneer. By the end, Actioneer will be able to query your datasets and read report data.

| | |
| :--- | :--- |
| &nbsp; **Prerequisites** | &nbsp; Power BI Pro or Premium workspace, Actioneer account |
| &nbsp; **Access granted** | &nbsp; Dataset queries and report reads via Fivetran sync |
| &nbsp; **Estimated time** | &nbsp; ~10 minutes |

<Info>
  Actioneer cannot create, edit, publish, or delete any reports, dashboards, or datasets in your Power BI workspace.
</Info>

## Connect via Fivetran

Actioneer uses [Fivetran](https://fivetran.com) to sync Power BI metadata and dataset information into your warehouse. Fivetran handles extraction and scheduling — Actioneer queries the synced tables.

<Steps>
  <Step title="Start the connection in Actioneer">
    1. In Actioneer, click **Data** in the left sidebar.
    2. Select **Microsoft Power BI** from the list of data sources.
    3. Enter a **Connection Name** — a descriptive label to identify this source (e.g., `Company Power BI`).
    4. Click **Connect**.
    5. Actioneer creates the warehouse connection and displays a **Connection Created** confirmation.
    6. Click **Continue to Power BI** — a new tab opens to the Fivetran configuration page.
  </Step>
  <Step title="Gather your Power BI credentials">
    Fivetran needs three values from your Microsoft account. Your Power BI or IT admin can provide these.

    | &nbsp; Field | &nbsp; Where to find it |
    | :--- | :--- |
    | &nbsp; **Tenant ID** | &nbsp; Your Microsoft Entra ID directory ID |
    | &nbsp; **Client ID** | &nbsp; The application ID of a registered app with Power BI read access |
    | &nbsp; **Client secret** | &nbsp; A valid secret for that registered app |

    <Tip>
      If you need to create these credentials from scratch, follow [Fivetran's Power BI setup guide](https://fivetran.com/docs/connectors/applications/microsoft-power-bi/setup-guide) — it walks through app registration and permissions step by step.
    </Tip>
  </Step>
  <Step title="Complete the Fivetran configuration">
    On the Fivetran configuration page, enter the three credentials:

    | &nbsp; Field | &nbsp; Value |
    | :--- | :--- |
    | &nbsp; **Tenant ID** | &nbsp; From Step 2 |
    | &nbsp; **Client ID** | &nbsp; From Step 2 |
    | &nbsp; **Client secret** | &nbsp; From Step 2 |

    Click **Save & Test**. Fivetran validates the connection and begins the initial sync.
  </Step>
</Steps>

## What syncs

Fivetran syncs the following Power BI metadata into your warehouse:

| &nbsp; Table | &nbsp; Description |
| :--- | :--- |
| &nbsp; **dataset** | &nbsp; Dataset metadata including name, configuration, and refresh schedules |
| &nbsp; **report** | &nbsp; Report definitions and workspace associations |
| &nbsp; **dashboard** | &nbsp; Dashboard metadata and tile layouts |
| &nbsp; **workspace** | &nbsp; Workspace details and membership |
| &nbsp; **refresh_history** | &nbsp; Dataset refresh logs and status |

Sync frequency is configured in Fivetran. Actioneer queries whatever lands in the warehouse.

## Common questions

<AccordionGroup>
  <Accordion title="Can Actioneer modify reports or dashboards in Power BI?">
    No. Actioneer reads synced metadata from your warehouse. It cannot create, edit, publish, or delete any reports, dashboards, or datasets in your Power BI workspace.
  </Accordion>

  <Accordion title="Where are my Power BI credentials stored?">
    Your Power BI credentials (Tenant ID, Client ID, and Client secret) are stored and managed by Fivetran, not by Actioneer. Fivetran encrypts all credentials at rest and in transit. Actioneer only queries the synced data in your warehouse.
  </Accordion>

  <Accordion title="What Power BI license do I need?">
    A Power BI Pro account is required at minimum. Power BI Premium provides additional features.
  </Accordion>
</AccordionGroup>
