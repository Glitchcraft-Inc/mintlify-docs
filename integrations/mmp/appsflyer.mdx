This guide walks through connecting an AppsFlyer account to Actioneer. By the end, Actioneer will be able to query your attribution data, campaign performance, and install events.

| At a glance | |
| :--- | :--- |
| **Prerequisites** | AppsFlyer account with **Admin** role and **Master API** enabled, Actioneer account |
| **Access granted** | Attribution data, campaign metrics, and install events via the AppsFlyer Pull API |
| **Estimated time** | ~10 minutes |

<Warning>
  Actioneer requires **Master API** access on your AppsFlyer account. Master API is a premium feature. If it is not enabled, contact your AppsFlyer account manager or email [partners@appsflyer.com](mailto:partners@appsflyer.com) to request access before proceeding.
</Warning>

<Info>
  Actioneer cannot create, modify, or delete any apps, partners, postbacks, or attribution settings in your AppsFlyer account.
</Info>

---

<Steps>
  <Step title="Retrieve your AppsFlyer API V2 token">
    AppsFlyer authenticates API access via an API V2 token (a bearer token) tied to your account. Only users with the **Admin** role can view and manage tokens.

    1. Sign in at [hq1.appsflyer.com](https://hq1.appsflyer.com).
    2. Open the account menu (your email address in the top-right corner) and select **Security Center**.
    3. In the **AppsFlyer API tokens** section, click **Manage your AppsFlyer API tokens** → locate **API V2.0**.
    4. If no token exists, click **Create token**. If one exists, copy it. If the token was generated before **March 10, 2026**, it has been revoked — click **Create token** to generate a new one.

| Value | Location | Example |
    | :--- | :--- | :--- |
| **API V2 Token** | Security Center → Manage your AppsFlyer API tokens → API V2.0 | `eyJhbGciOiJBMjU2S1ci...` |

    <Warning>
      The API V2 token grants read access to your AppsFlyer data across all apps. Treat it like a password: do not share it over unencrypted channels or commit it to version control. AppsFlyer recommends rotating tokens every 180 days.
    </Warning>

    <Note>
      **IP allowlist:** If your AppsFlyer account has an API allowlist enabled (Security Center → API allowlist), only requests from whitelisted IP addresses can use V2 token APIs. Add Actioneer's IP address `35.244.14.238` to the allowlist before proceeding. Allowlist management is available to Admins only.
    </Note>
  </Step>

  <Step title="Connect AppsFlyer in Actioneer">
    1. In Actioneer, click **Data** in the left sidebar.
    2. Select **AppsFlyer** from the list of data sources.
    3. When prompted, select **Direct Connection**.
    4. Complete the connection form:

| Field | Value |
    | :--- | :--- |
| **Connection Name** | A descriptive label, e.g. `Production AppsFlyer` |
| **API V2 Token** | From Step 1 |

    5. Click **Connect**.
  </Step>

  <Step title="Verify and select apps">
    After connecting, Actioneer verifies your API V2 token and retrieves the list of apps available in your AppsFlyer account.

    1. Review the list of available apps.
    2. Select the apps whose data Actioneer should query.
    3. Confirm your selection.

    <Info>
      You can update the selected apps later from the connection settings.
    </Info>
  </Step>
</Steps>

## Common questions

<AccordionGroup>
  <Accordion title="What AppsFlyer data can Actioneer access?">
    Actioneer can query attribution data, campaign performance, install events, in-app events, and cohort data accessible via the AppsFlyer Pull API. It cannot modify partners, postbacks, attribution settings, or any account configuration.
  </Accordion>

  <Accordion title="What if my account has an API allowlist enabled?">
    If your AppsFlyer account enforces an API allowlist, add `35.244.14.238` to the allowed IP addresses under Security Center → API allowlist. Without this, API calls from Actioneer will be rejected. The allowlist applies to all APIs that use the V2.0 token.
  </Accordion>

  <Accordion title="How does Actioneer protect credentials?">
    All credentials are encrypted at rest (AES-256) and in transit (TLS 1.3); Actioneer is SOC 2 Type II and ISO 27001 certified (audited by Schellman), and GDPR compliant. See [Access & networking](/operations/access-and-networking).
  </Accordion>
</AccordionGroup>
