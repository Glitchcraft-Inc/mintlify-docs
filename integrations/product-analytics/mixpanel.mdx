# Linking Mixpanel as a Source

This guide walks through exporting Mixpanel data to a BigQuery dataset and connecting that dataset to Actioneer. Mixpanel's **Data Pipelines** feature handles the export — Actioneer queries the resulting tables.

| &nbsp; | &nbsp; |
| :--- | :--- |
| &nbsp; **Prerequisites** | &nbsp; Mixpanel account on a **Growth** or **Enterprise** plan with the Data Pipelines add-on, Google Cloud project with BigQuery enabled, Actioneer account |
| &nbsp; **Access granted** | &nbsp; Events, user profiles, and identity mappings via BigQuery |
| &nbsp; **Estimated time** | &nbsp; ~20–30 minutes |

<Info>
  Actioneer never connects to Mixpanel directly. It reads from the BigQuery dataset where Mixpanel exports your data.
</Info>

---

<Steps>
  <Step title="Create a BigQuery dataset for Mixpanel exports">
    Mixpanel needs a dedicated BigQuery dataset to write exported data into.

    1. Open the [Google Cloud Console](https://console.cloud.google.com) and select your project.
    2. Go to **BigQuery** in the left sidebar.
    3. Click the three-dot menu next to your project name and select **Create dataset**.
    4. Configure the dataset:

| &nbsp; Field | &nbsp; Value |
    | :--- | :--- |
| &nbsp; **Dataset ID** | &nbsp; A descriptive name, e.g. `mixpanel_export` |
| &nbsp; **Location** | &nbsp; Must match one of Mixpanel's [supported regions](https://docs.mixpanel.com/docs/data-pipelines/integrations/bigquery) |

    5. Click **Create dataset**.

    <Note>
      If your GCP organization uses domain restriction constraints, you must allowlist `mixpanel.com` and Google Workspace customer ID `C00m5wrjz` before proceeding.
    </Note>
  </Step>

  <Step title="Grant Mixpanel access to your dataset">
    Mixpanel exports data using its own service account — you do not create one yourself. You grant this service account access to your dataset.

    **Grant BigQuery Job User (project-level):**

    1. In the Google Cloud Console, go to **IAM & Admin** → **IAM**.
    2. Click **Grant Access**.
    3. In the **New principals** field, enter: `export-upload@mixpanel-prod-1.iam.gserviceaccount.com`
    4. Assign the role **BigQuery Job User**.
    5. Click **Save**.

    **Grant BigQuery Data Owner (dataset-level):**

    1. In BigQuery, click your export dataset.
    2. Click **Sharing** → **Permissions** → **Add Principal**.
    3. Enter the same service account: `export-upload@mixpanel-prod-1.iam.gserviceaccount.com`
    4. Assign the role **BigQuery Data Owner**.
    5. Click **Save**.

| &nbsp; Role | &nbsp; Scope | &nbsp; Purpose |
    | :--- | :--- | :--- |
| &nbsp; **BigQuery Job User** | &nbsp; Project-level (IAM) | &nbsp; Run export jobs |
| &nbsp; **BigQuery Data Owner** | &nbsp; Dataset-level (Sharing → Permissions) | &nbsp; Create and write tables in the export dataset |

    <Warning>
      Mixpanel requires **BigQuery Data Owner** on the dataset, not Data Editor. This is because Mixpanel manages table schemas and partitioning during export.
    </Warning>
  </Step>

  <Step title="Create the pipeline in Mixpanel">
    1. Sign in to [mixpanel.com](https://mixpanel.com) and open your project.
    2. Go to **Data Pipelines** → **Integrations** → **BigQuery**.
    3. Complete the pipeline form:

| &nbsp; Field | &nbsp; Value |
    | :--- | :--- |
| &nbsp; **GCP Project ID** | &nbsp; Your Google Cloud project ID |
| &nbsp; **Dataset Name** | &nbsp; The dataset from Step 1 (e.g. `mixpanel_export`) |
| &nbsp; **GCP Region** | &nbsp; The region you selected when creating the dataset |

    4. Click **Create Pipeline**.

    <Info>
      The first export may take several hours depending on your data volume. Subsequent exports run automatically on a recurring schedule.
    </Info>
  </Step>

  <Step title="Connect the BigQuery dataset in Actioneer">
    Once the first Mixpanel export completes and tables appear in BigQuery, connect the dataset to Actioneer.

    1. In Actioneer, click **Data** in the left sidebar.
    2. Select **Google BigQuery** from the list of data sources.
    3. When prompted, select **Direct Connection**.
    4. Complete the connection form using the same project and dataset where Mixpanel exports data.

    For full connection instructions, see the [Google BigQuery guide](/integrations/data-warehouses/google-big-query).

    <Warning>
      The Actioneer service account only needs **read** access (`BigQuery Data Viewer` + `BigQuery Job User`) to the Mixpanel export dataset. Do not reuse Mixpanel's service account for Actioneer.
    </Warning>
  </Step>
</Steps>

## Exported tables

Mixpanel creates the following tables in your dataset:

| &nbsp; Table | &nbsp; Contains |
| :--- | :--- |
| &nbsp; `mp_master_event` | &nbsp; All events, with properties stored as JSON |
| &nbsp; `mp_people_data_view` | &nbsp; User profiles (use this view, not the underlying tables) |
| &nbsp; `mp_identity_mappings_data_view` | &nbsp; Identity mappings (use this view, not the underlying tables) |
| &nbsp; `mp_nessie_export_log` | &nbsp; Export audit log |

<Warning>
  Do not modify the schema of tables created by Mixpanel, and do not change the `TIMEPARTITIONING` settings on any table. Either change will cause the export pipeline to fail.
</Warning>

<Tip>
  Always query the `_view` suffixed objects for user profiles and identity mappings — not the underlying partitioned tables. When querying identity mappings, use `resolved_distinct_id` rather than `distinct_id`.
</Tip>

## Common questions

<AccordionGroup>
  <Accordion title="What Mixpanel data can Actioneer access?">
    Actioneer can query any tables Mixpanel writes to the BigQuery dataset — events, user profiles, and identity mappings. The exact tables depend on your Mixpanel project configuration.
  </Accordion>

  <Accordion title="Does Actioneer connect to Mixpanel directly?">
    No. Actioneer only reads from BigQuery. Mixpanel handles the export independently. Actioneer cannot access, modify, or configure anything in your Mixpanel account.
  </Accordion>

  <Accordion title="What if my GCP organization uses VPC Service Controls?">
    IP allowlists are not supported for Mixpanel's export pipeline. If your organization enforces VPC Service Controls, contact Mixpanel support for guidance on configuring access.
  </Accordion>

  <Accordion title="How does Actioneer protect credentials?">
    All credentials are encrypted at rest with AES-256 and encrypted in transit with TLS 1.3. Each customer's credentials are stored in isolation and are never shared across accounts or logged in plain text. Actioneer is SOC 2 Type II certified, ISO 27001 certified (audited by Schellman), and GDPR compliant.
  </Accordion>
</AccordionGroup>
