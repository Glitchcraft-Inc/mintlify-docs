# Linking Amplitude as a Source

This guide walks through exporting Amplitude data to a BigQuery dataset and connecting that dataset to Actioneer. Amplitude's **BigQuery Destination** handles the export — Actioneer queries the resulting tables.

| &nbsp; | &nbsp; |
| :--- | :--- |
| &nbsp; **Prerequisites** | &nbsp; Amplitude account (Growth or Enterprise plan), Google Cloud project with BigQuery enabled, Actioneer account |
| &nbsp; **Access granted** | &nbsp; Raw events, user properties, and identity data via BigQuery |
| &nbsp; **Estimated time** | &nbsp; ~20–30 minutes |

<Info>
  Actioneer never connects to Amplitude directly. It reads from the BigQuery dataset where Amplitude exports your data.
</Info>

---

<Steps>
  <Step title="Create a BigQuery dataset for Amplitude exports">
    Amplitude needs a dedicated BigQuery dataset to write exported data into.

    1. Open the [Google Cloud Console](https://console.cloud.google.com) and select your project.
    2. Go to **BigQuery** in the left sidebar.
    3. Click the three-dot menu next to your project name and select **Create dataset**.
    4. Configure the dataset:

| &nbsp; Field | &nbsp; Value |
    | :--- | :--- |
| &nbsp; **Dataset ID** | &nbsp; A descriptive name, e.g. `amplitude_export` |
| &nbsp; **Location** | &nbsp; Choose the region closest to your Amplitude data residency |

    5. Click **Create dataset**.
  </Step>

  <Step title="Create a GCP service account for Amplitude">
    Amplitude authenticates to BigQuery using a GCP service account. The account needs write access to your export dataset.

    1. In the Google Cloud Console, go to **IAM & Admin** → **Service Accounts**.
    2. Click **Create Service Account**.
    3. Name it `amplitude-export` (or similar) and click **Create and Continue**.
    4. Grant two roles:

| &nbsp; Role | &nbsp; Scope | &nbsp; Purpose |
    | :--- | :--- | :--- |
| &nbsp; **BigQuery Data Editor** | &nbsp; The export dataset | &nbsp; Write exported tables |
| &nbsp; **BigQuery Job User** | &nbsp; Project-level | &nbsp; Run export jobs |

    5. Click **Done**, then click into the new service account.
    6. Go to the **Keys** tab → **Add Key** → **Create new key** → **JSON**.
    7. Save the downloaded JSON key file securely.

    <Warning>
      This JSON key grants write access to your BigQuery dataset. Store it in a password manager and do not commit it to version control.
    </Warning>
  </Step>

  <Step title="Configure the BigQuery Destination in Amplitude">
    1. Sign in to [analytics.amplitude.com](https://analytics.amplitude.com) and open your project.
    2. Go to **Data** → **Destinations**.
    3. Search for **BigQuery** and select it.
    4. Complete the destination form:

| &nbsp; Field | &nbsp; Value |
    | :--- | :--- |
| &nbsp; **GCP Project ID** | &nbsp; Your Google Cloud project ID |
| &nbsp; **Dataset** | &nbsp; The dataset from Step 1 (e.g. `amplitude_export`) |
| &nbsp; **Service Account Key** | &nbsp; Upload the JSON key from Step 2 |
| &nbsp; **Data to export** | &nbsp; Select the data types you need (events, user properties, merged identities) |
| &nbsp; **Sync frequency** | &nbsp; Hourly or daily (depending on your Amplitude plan) |

    5. Click **Save** to activate the destination.

    <Note>
      The BigQuery Destination is available on Amplitude's **Growth** and **Enterprise** plans. If you do not see it under Destinations, contact your Amplitude account manager.
    </Note>

    <Info>
      The first sync may take several hours to backfill historical data. Subsequent syncs run on the schedule you selected. Amplitude creates date-partitioned tables in your dataset.
    </Info>
  </Step>

  <Step title="Connect the BigQuery dataset in Actioneer">
    Once the first Amplitude sync completes and tables appear in BigQuery, connect the dataset to Actioneer.

    1. In Actioneer, click **Data** in the left sidebar.
    2. Select **Google BigQuery** from the list of data sources.
    3. When prompted, select **Direct Connection**.
    4. Complete the connection form using the same project and dataset where Amplitude exports data.

    For full connection instructions, see the [Google BigQuery guide](/integrations/data-warehouses/google-big-query).

    <Warning>
      The Actioneer service account only needs **read** access (`BigQuery Data Viewer` + `BigQuery Job User`) to the Amplitude export dataset. Do not reuse the Amplitude service account — create a separate one for Actioneer with read-only permissions.
    </Warning>
  </Step>
</Steps>

## Common questions

<AccordionGroup>
  <Accordion title="What Amplitude data can Actioneer access?">
    Actioneer can query any tables Amplitude writes to the BigQuery dataset — raw events, merged user properties, and identity data. The exact tables depend on which data types you selected when configuring the destination.
  </Accordion>

  <Accordion title="How often is the data updated?">
    Amplitude syncs data on the schedule you configure — typically hourly or daily depending on your plan. Actioneer queries the latest data available in BigQuery at the time of each query.
  </Accordion>

  <Accordion title="Does Actioneer connect to Amplitude directly?">
    No. Actioneer only reads from BigQuery. Amplitude handles the export independently. Actioneer cannot access, modify, or configure anything in your Amplitude account.
  </Accordion>

  <Accordion title="How does Actioneer protect credentials?">
    All credentials are encrypted at rest with AES-256 and encrypted in transit with TLS 1.3. Each customer's credentials are stored in isolation and are never shared across accounts or logged in plain text. Actioneer is SOC 2 Type II certified, ISO 27001 certified (audited by Schellman), and GDPR compliant.
  </Accordion>
</AccordionGroup>
