The IP to allowlist, the base URLs, how authentication works, and how Actioneer protects credentials.

## Egress IP

Actioneer connects to your data sources from a single, fixed egress IP. Allowlist it wherever a source supports IP restrictions.

| | |
| :--- | :--- |
| **Actioneer egress IP** | `35.244.14.238` |

<Tip>
  Restricting source access to `35.244.14.238/32` is a second layer of defense on top of least-privilege credentials. The [Snowflake guide](/integrations/data-warehouses/snowflake) shows it applied as a network policy.
</Tip>

## Base URLs

The ingestion API is served over HTTPS:

| Environment | Base URL |
| :--- | :--- |
| **Production** | `https://api.actioneer.com` |
| **Staging** | A distinct host, confirmed during onboarding |

Production and staging are separate hosts — never point production traffic at staging or vice versa.

## Authentication model

| Surface | How it authenticates |
| :--- | :--- |
| **Data sources** | A dedicated, least-privilege service identity you create per source (e.g. a service user, service account, or read-only API token), scoped to read access only. |
| **Ingestion API** | An `X-API-Key` request header carrying a server-side secret. One key authenticates every envelope in a `/batch` request. |

<Warning>
  The ingestion API key is a server-side secret. Issue requests from a backend or trusted server context — never embed it in distributable client binaries or web bundles. Never place secrets, credentials, tokens, or cardholder data inside event or profile properties.
</Warning>

## Least privilege & read-only

Every data-source connection is scoped to the minimum access required to read the data you select. Actioneer cannot create, modify, or delete tables, schemas, or source data. Where a service requires write access for its own operation (for example, Athena writing temporary query results to S3), permissions are scoped strictly to that purpose.

## Credential protection

All credentials are encrypted at rest with AES-256 and in transit with TLS 1.3. They are never stored in plain text or logged. Actioneer is SOC 2 Type II certified, ISO 27001 certified (audited by Schellman), and GDPR compliant.
