# Knowledge

Knowledge is the memory layer that makes Actioneer different from a generic AI query tool. It is a persistent, structured store of your organisation's business context — metric definitions, KPI targets, past campaigns, team preferences, and validated playbooks. Every interaction with the platform adds to it.

---

## What Knowledge Contains

| Context Type | Examples |
| :--- | :--- |
| **Metric definitions** | "D7 Retention" means users who returned within 7 days of first open. "Conversion" means completed KYC within 14 days of signup. |
| **KPI targets** | Monthly active users target: 500K. First-investment conversion target: 12%. |
| **Campaign history** | "Spring Re-engagement" ran in March, targeted dormant users, achieved 8% reactivation. |
| **Playbooks** | Weekly retention analysis methodology — which tables, which filters, which output format. |
| **Schema context** | Table relationships, column meanings, event taxonomy, data freshness schedules. |
| **Team preferences** | Default breakdowns (by city, by platform), preferred chart types, reporting cadence. |

---

## How Knowledge Is Built

| Channel | Description |
| :--- | :--- |
| **Onboarding** | Actioneer's team runs a discovery process. Metric definitions, KPI targets, schema context, and playbooks are mapped from your existing BI tools, documentation, and conversations with your data team. |
| **Usage** | Every interaction refines the Knowledge layer. Corrections, adjusted preferences, and validated Playbooks are stored at the user or organisation level. |
| **Automated learning** | Actioneer detects patterns across queries — commonly requested dimensions, frequently compared segments, recurring analysis patterns — and incorporates them. |

---

## Knowledge Tiers

Context is stored at two levels to balance consistency and personalisation.

| Tier | Scope | Examples |
| :--- | :--- | :--- |
| **Organisation** | Shared across the entire team | Metric definitions, KPI targets, schema mappings |
| **User** | Specific to an individual | Preferred chart types, default breakdowns, past questions |

Organisation-level knowledge ensures consistency — everyone gets the same answer for the same question. User-level knowledge ensures personalisation — results are formatted the way each person prefers.

---

## Why Knowledge Matters

Without business context, an AI tool is guessing. It might generate valid SQL, but if it defines "retention" differently from how your team defines it, the answer is wrong even if the query is correct.

Knowledge eliminates this class of error. It grounds every answer in your definitions, not the model's assumptions. The more Actioneer is used, the more precise it becomes — a compounding advantage that generic query tools cannot replicate.

<Tip>
  Knowledge is versioned and auditable. When a metric definition changes, the update is tracked with a timestamp and the user who made the change.
</Tip>
