# Knowledge

The Knowledge Base is a store of context — metric definitions, business rules, benchmarks, and data validation notes — that Actioneer references during every analysis. It makes answers more accurate, consistent, and tailored to your specific business.

---

## How Knowledge Improves Accuracy

| Scenario | Without Knowledge | With Knowledge |
| :--- | :--- | :--- |
| **CPI calculation** | Generic formula: total spend / total installs | Your definition: paid spend only / attributed installs excluding organics |
| **Launch context** | Treats all dates equally | Knows July 29th was global launch — excludes pre-launch data from benchmarks |
| **Data quality** | May query unreliable tables | Knows to never use `cost_etl_*` tables — they contain erroneous data |
| **Evaluation** | No reference point for "good" or "bad" | Knows industry D1 retention for casual games is 35% |

Knowledge turns generic analysis into *your* analysis.

---

## Knowledge Types

| Type | What it does | Example |
| :--- | :--- | :--- |
| **Metric** (`METRIC`) | Ensures consistent formulas across all analyses | *"ARPDAU = IAP revenue only, excludes ad revenue"* |
| **Metric Range** (`METRIC_RANGE`) | Flags when a value looks abnormal | *"CPI should be $0.50–$5.00; flag if outside this range"* |
| **Insight** (`INSIGHT`) | Provides business context that can't be inferred from data | *"July 29th was global launch — exclude pre-launch data"* |
| **Data Validation** (`DATA_VALIDATION`) | Prevents use of bad tables or wrong filters | *"Never use cost_etl_* tables — erroneous data"* |
| **External Benchmark** (`EXTERNAL_BENCHMARK`) | Gives reference points for evaluation | *"Industry D1 retention for casual games is 35%"* |
| **Reporting** (`REPORTING`) | Standardises output format | *"Always report revenue in USD, rounded to nearest dollar"* |
| **Segment** (`SEGMENT`) | Consistent audience definitions | *"Whales = users with LTV > $100 in first 30 days"* |
| **Visualisation** (`VISUALISATION`) | Charting preferences and standards | *"Use line charts for time-series, bar charts for comparisons"* |

## Priority and Scope

**Priority levels:**

| Priority | When to use |
| :--- | :--- |
| **Critical** | Must always be applied — e.g. *"Never use cost_etl tables"* |
| **High** | Applied by default — e.g. metric definitions, key business context |
| **Low** | Referenced when relevant — e.g. nice-to-have benchmarks |

**Scope levels:**

| Level | Visibility |
| :--- | :--- |
| **Tenant** | Shared across all users in your organisation |
| **User** | Private to your account only |

## Adding Knowledge

### Text-based / manual entry

Tell Actioneer what to remember in plain language:
- *"Add knowledge: D1 retention benchmark for hyper-casual games is 40%"*
- *"Save this rule: always filter acquisition data by tenant_id = 'Spektra'"*

Actioneer asks you to confirm the type, priority, and level before saving.

### File-based imports

Upload documents and Actioneer indexes their content.

| Format | Use case |
| :--- | :--- |
| **PDF** | Reports, strategy docs, benchmark studies |
| **CSV** | Metric lookup tables, reference data |
| **Word (.docx)** | Business documentation |
| **Text (.txt)** | Plain text notes |

Uploaded files are searchable — relevant passages are retrieved during analysis.

### Integrations

Connect external sources to keep knowledge current:
- **Web research** — search for industry benchmarks and save findings
- **Internal wikis** — upload exports from Confluence, Notion, or similar
- **Previous analyses** — save key findings from past work as reusable context

### From analysis results

After any analysis, Actioneer may suggest saving a finding:
- *"Save this insight: US has 2x higher ARPU than other markets"*

## Managing Knowledge

| Action | How to do it | Example |
| :--- | :--- | :--- |
| **Search** | Ask what Actioneer knows about a topic | *"What do you know about retention?"* |
| **List all** | Browse the full knowledge base | *"List all knowledge entries"* |
| **Filter** | View by type or priority | *"Show all CRITICAL knowledge"* |
| **Delete** | Remove a specific entry — confirmation required | *"Delete knowledge entry about CPI benchmark"* |

<Info>
  Knowledge is versioned and auditable. When a definition changes, the update is tracked with a timestamp and the user who made the change.
</Info>
