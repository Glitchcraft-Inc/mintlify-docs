# Boards

Boards are persistent, organised pages where findings from Chat, Playbooks, and other analyses can be saved, revisited, and compared. Think of them as a dashboard you build through conversation — not through a drag-and-drop chart editor.

---

## Board Structure

A Board is a scrollable page made of sections. Each section contains a grid of cards.

| Element | Description |
| :--- | :--- |
| **Section** | A named group of related cards. Collapsible, reorderable, and renamable. |
| **Card** | A single visualisation: line chart, bar chart, table, or metric number. |
| **Insight line** | A short annotation attached to a section summarising the takeaway. |

---

## Creating a Board

| Method | Description |
| :--- | :--- |
| **From a template** | Pick a template — Product KPIs, Growth, Revenue. Actioneer reads your schema and auto-generates sections with relevant charts. |
| **From a research report** | After a deep research conversation in Chat, save the findings as a Board. Each investigated area becomes a section. |
| **From scratch** | Start with a blank Board and describe what each card should show. Actioneer generates the queries automatically. |
| **From a business deck** | Upload a presentation PDF. Each slide becomes a section, charts become cards, commentary becomes paired text cards. |
| **By pinning** | Any result in Chat, Funnels, Retention, or Segments can be pinned to a Board with one click. |

---

## Card Types

Actioneer auto-detects the appropriate visualisation based on the shape of the result.

| Result Shape | Default Card Type |
| :--- | :--- |
| Time series | Line chart |
| Single number | Metric card |
| Ranked list | Bar chart |
| Tabular data | Table |

---

## Board Management

| Action | Description |
| :--- | :--- |
| **Rename** | Rename a Board, section, or card at any time |
| **Reorder** | Drag sections into place |
| **Global date filter** | Update all time-based cards at once |
| **Delete** | Remove individual cards or entire sections without affecting other parts |

<Info>
  Boards are scoped to the active dataset. If you connect multiple data sources, each Board is associated with the dataset it was created against.
</Info>
