# Retention

Retention in Actioneer tracks whether users return after their initial action — and how that behaviour changes over time, across cohorts, and between Segments. It answers the fundamental question: are users sticking around?

---

## Retention Components

| Component | Description |
| :--- | :--- |
| **Starting event** | The action that defines when a user enters the cohort — e.g. "First App Open" |
| **Return event** | The action that counts as a return — e.g. "Any Session" or "Made a Purchase" |
| **Cohort window** | The time period used to group users — daily, weekly, or monthly |
| **Retention curve** | The percentage of the cohort that returned at each subsequent interval |

---

## Retention Views

| View | Description |
| :--- | :--- |
| **Cohort table** | A matrix showing retention rates over time. Rows are cohorts (by signup week). Columns are intervals (Day 1, Day 7, Day 30). Each cell shows the percentage that returned. |
| **Retention curve** | A line chart plotting retention over time for one or more cohorts. Useful for spotting whether retention is improving or declining. |
| **Segment comparison** | Compare retention between two or more Segments — paid vs organic, by region, by acquisition channel. |

---

## Conditional Cohorts

Standard retention groups users by a single event. Actioneer also supports conditional cohort definitions — adding criteria to the starting event that narrow the population.

| Condition | Example |
| :--- | :--- |
| **Event + time constraint** | Users who signed up and completed KYC within 48 hours |
| **Event + value filter** | Users who made a first purchase where order value exceeded ₹1,000 |
| **Event + engagement threshold** | Users from a specific acquisition channel who engaged with at least 3 sessions in their first week |

This lets you compare retention between meaningfully different user groups, not just calendar-based cohorts.

---

## Acting on Retention Data

| Action | Description |
| :--- | :--- |
| **Save a cohort as a Segment** | Target users who retained at Day 7 but churned by Day 30 for a winback campaign |
| **Pin to a Board** | Add the retention curve to a persistent dashboard for ongoing monitoring |
| **Feed into a Playbook** | Schedule a retention analysis to run weekly and auto-deliver a graded report |
| **Set an alert** | Get notified when retention for a specific cohort drops below a threshold |

<Info>
  Actioneer processes at least 12 months of retention data in hot storage for forecasting and trend analysis. Older data is available in cold storage for historical lookups.
</Info>
