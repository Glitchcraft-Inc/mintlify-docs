# Retention

Acquisition means nothing if users don't come back. **Retention tells you who stays — and who doesn't.**

Retention tracks how many users return to your app after their initial install or first activity. It shows user engagement over time using cohort-based retention curves.

---

## What Is a Retention View

A Retention View is a saved analysis showing the percentage of users who return at defined intervals after their starting event.

**Key terms:**

| &nbsp; Term | &nbsp; Meaning |
| :--- | :--- |
| &nbsp; **D1 Retention** | &nbsp; % of users who return 1 day after install |
| &nbsp; **D7 Retention** | &nbsp; % of users who return 7 days after install |
| &nbsp; **D30 Retention** | &nbsp; % of users who return 30 days after install |
| &nbsp; **Cohort** | &nbsp; Group of users who installed on the same date, week, or month |
| &nbsp; **Retention event** | &nbsp; The action that counts as "returning" — e.g. app open, screen view, any user-triggered event |

---

## Retention Components

| &nbsp; Component | &nbsp; Description |
| :--- | :--- |
| &nbsp; **Starting event** | &nbsp; The action that defines when a user enters the cohort — e.g. "First App Open" |
| &nbsp; **Return event** | &nbsp; The action that counts as a return — e.g. "Any Session" or "Made a Purchase" |
| &nbsp; **Cohort window** | &nbsp; The time period used to group users — daily, weekly, or monthly |
| &nbsp; **Retention curve** | &nbsp; The percentage of the cohort that returned at each subsequent interval |

---

## Retention Views

| &nbsp; View | &nbsp; Description |
| :--- | :--- |
| &nbsp; **Retention table** | &nbsp; Grid showing % of users retained at D1, D3, D7, D14, D30 by install cohort |
| &nbsp; **Retention curves** | &nbsp; Visual trend lines showing how retention decays over time |
| &nbsp; **Cohort comparison** | &nbsp; Compare retention across different install weeks or months |
| &nbsp; **Segment comparison** | &nbsp; Compare retention between Segments — paid vs organic, by region, by channel |

Example: *"Show me D1, D7, and D30 retention for users who installed in April 2026, broken down by country"*

---

## Conditional Cohorts

Add criteria to the starting event that narrow the population:

| &nbsp; Condition | &nbsp; Example |
| :--- | :--- |
| &nbsp; **Event + time constraint** | &nbsp; Users who signed up and completed KYC within 48 hours |
| &nbsp; **Event + value filter** | &nbsp; Users who made a first purchase where order value exceeded ₹1,000 |
| &nbsp; **Event + engagement threshold** | &nbsp; Users from a specific channel who had at least 3 sessions in their first week |

---

## Managing Retention

| &nbsp; Action | &nbsp; How to do it | &nbsp; Example prompt |
| :--- | :--- | :--- |
| &nbsp; **View retention** | &nbsp; Ask for specific metrics and cohorts | &nbsp; *"Show me D1, D7, D30 retention for last month"* |
| &nbsp; **Break down** | &nbsp; Split by dimension | &nbsp; *"Retention by country/platform/campaign"* |
| &nbsp; **Compare cohorts** | &nbsp; Overlay different time periods | &nbsp; *"Compare March vs April retention"* |
| &nbsp; **Pin to a Board** | &nbsp; Attach for ongoing monitoring — refreshes automatically | &nbsp; *"Pin retention to my engagement board"* |
| &nbsp; **Delete** | &nbsp; Remove a saved Retention view — confirmation required | &nbsp; *"Delete the April retention view"* |
