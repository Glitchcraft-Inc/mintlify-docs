# Retention

Acquisition means nothing if users don't come back. **Retention tells you who stays — and who doesn't.**

Retention tracks how many users return to your app after their initial install or first activity. It shows user engagement over time using cohort-based retention curves.

---

## What Is a Retention View

A Retention View is a saved analysis showing the percentage of users who return at defined intervals after their starting event.

**Key terms:**

| Term | Meaning |
| :--- | :--- |
| **D1 Retention** | % of users who return 1 day after install |
| **D7 Retention** | % of users who return 7 days after install |
| **D30 Retention** | % of users who return 30 days after install |
| **Cohort** | Group of users who installed on the same date, week, or month |
| **Retention event** | The action that counts as "returning" — e.g. app open, screen view, any user-triggered event |

---

## Retention Components

| Component | Description |
| :--- | :--- |
| **Starting event** | The action that defines when a user enters the cohort — e.g. "First App Open" |
| **Return event** | The action that counts as a return — e.g. "Any Session" or "Made a Purchase" |
| **Cohort window** | The time period used to group users — daily, weekly, or monthly |
| **Retention curve** | The percentage of the cohort that returned at each subsequent interval |

---

## Retention Views

| View | Description |
| :--- | :--- |
| **Retention table** | Grid showing % of users retained at D1, D3, D7, D14, D30 by install cohort |
| **Retention curves** | Visual trend lines showing how retention decays over time |
| **Cohort comparison** | Compare retention across different install weeks or months |
| **Segment comparison** | Compare retention between Segments — paid vs organic, by region, by channel |

Example: *"Show me D1, D7, and D30 retention for users who installed in April 2026, broken down by country"*

---

## Conditional Cohorts

Add criteria to the starting event that narrow the population:

| Condition | Example |
| :--- | :--- |
| **Event + time constraint** | Users who signed up and completed KYC within 48 hours |
| **Event + value filter** | Users who made a first purchase where order value exceeded ₹1,000 |
| **Event + engagement threshold** | Users from a specific channel who had at least 3 sessions in their first week |

---

## Managing Retention

| Action | How to do it | Example prompt |
| :--- | :--- | :--- |
| **View retention** | Ask for specific metrics and cohorts | *"Show me D1, D7, D30 retention for last month"* |
| **Break down** | Split by dimension | *"Retention by country/platform/campaign"* |
| **Compare cohorts** | Overlay different time periods | *"Compare March vs April retention"* |
| **Pin to a Board** | Attach for ongoing monitoring — refreshes automatically | *"Pin retention to my engagement board"* |
| **Delete** | Remove a saved Retention view — confirmation required | *"Delete the April retention view"* |
