# Metrics

Metrics in Actioneer are centrally defined, governed business measures. They ensure that when someone asks about "conversion" or "retention," the answer reflects what those terms actually mean at your company — not a generic assumption.

---

## Why Centralised Metrics Matter

In most organisations, the same metric is defined differently across tools. Retention in the BI dashboard might use a 7-day window. Retention in the CDP might use 30 days. The analyst's ad-hoc query might use yet another definition.

Actioneer solves this by maintaining a single, authoritative set of metric definitions that every feature on the platform uses — Chat, Boards, Playbooks, Funnels, and Segments all reference the same source of truth.

---

## Metric Definition

Each metric in Actioneer includes the following components.

| Component | Description |
| :--- | :--- |
| **Name** | A human-readable label, e.g. "D7 Retention" or "First Investment Conversion" |
| **Definition** | The precise business logic — what counts, what does not, and over what time window |
| **SQL Mapping** | The underlying query that computes the metric from your warehouse tables |
| **Dimensions** | The breakdowns the metric supports — by city, channel, platform, cohort |
| **Owner** | The team or individual responsible for the definition |

<Info>
  Metric definitions are version-controlled. When a definition changes, Actioneer tracks the update so historical comparisons remain valid.
</Info>

---

## Where Metrics Are Used

Once defined, metrics are available across the entire platform.

| Surface | How metrics are used |
| :--- | :--- |
| **Chat** | Ask a question referencing a metric by name. Actioneer uses the defined logic. |
| **Boards** | Add a metric card to a Board. It always computes using the governed definition. |
| **Playbooks** | Reference metrics in automated workflows. The same definition runs on every execution. |
| **Funnels** | Conversion steps can reference metric-defined thresholds. |
| **Alerts** | Monitor a metric and trigger a notification when it crosses a boundary. |

---

## Metric Governance

Metric definitions are managed at the organisation level. Changes are tracked and auditable.

| Governance feature | Description |
| :--- | :--- |
| **Change tracking** | All updates are logged with a timestamp and the user who made the change |
| **Historical preservation** | Values computed under a previous definition are preserved — not retroactively recalculated |
| **Conflict detection** | Duplicate or conflicting definitions are flagged during setup |

<Tip>
  During onboarding, Actioneer's team runs a discovery process to map your existing metric definitions from BI tools, spreadsheets, and documentation.
</Tip>
