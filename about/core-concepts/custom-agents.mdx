# Custom Agents

Custom Agents are bespoke agentic capabilities built to the specific operational shape of an organisation. Where Conversational Analytics, Scouts, and Playbooks cover common patterns, Custom Agents handle the workflows unique to your business.

---

## What Custom Agents Do

| Use Case | Description |
| :--- | :--- |
| **Variance commentary** | Automatically generate narrative explanations for period-over-period changes in key metrics |
| **Cross-system reconciliation** | Reconcile data across systems — warehouse vs CRM vs billing — and flag discrepancies |
| **Exception summarisation** | Summarise exceptions, outliers, and edge cases across a dataset into actionable briefs |
| **Threshold breach handling** | When a metric breaches a defined threshold, execute a pre-defined response sequence |
| **Period close assistance** | Automate the analytical steps of month-end or quarter-end close processes |

---

## How Custom Agents Are Built

| Phase | Description |
| :--- | :--- |
| **Scoping** | The operational workflow is documented — inputs, steps, decisions, outputs, and edge cases |
| **Configuration** | The agent is built on the Agent Orchestration Runtime — connected to the relevant data sources and governed by the Context Engine |
| **Testing** | Validated against historical data and real scenarios before production deployment |
| **Deployment** | Shipped to production with observability, audit logging, and RBAC from day one |

---

## Governance

Custom Agents are governed by the same controls as the rest of the platform.

| Control | Description |
| :--- | :--- |
| **Observability** | Trace-level visibility into every decision the agent makes |
| **RBAC** | Who can build, run, and view the agent's outputs is controlled at the workspace level |
| **Audit trail** | Every run is logged — inputs, outputs, model calls, and execution time |
| **Confidence flagging** | Low-confidence outputs are flagged rather than acted on |
