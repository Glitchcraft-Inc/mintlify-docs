# Playbooks

A Playbook is a validated analytical workflow — a sequence of queries, analysis steps, and output formats — that has been frozen so it can be rerun reliably with fresh data. Playbooks turn one-off analysis into repeatable process.

---

## Why Playbooks Exist

Ad-hoc analysis is not reproducible. The same question asked on two different days can produce different SQL, different filters, and different conclusions. If an analyst leaves or a metric definition shifts, the methodology is lost.

Playbooks capture the full methodology — not just the final answer, but every step, validation check, and output format — so it can be executed consistently, by anyone, at any time.

---

## Playbook Components

| Component | Description |
| :--- | :--- |
| **Steps** | Ordered sequence of SQL queries, analysis operations, and validation checks |
| **Parameters** | Configurable inputs — date ranges, segments, thresholds — that can change between runs |
| **Branching logic** | Conditional paths that execute based on intermediate results |
| **Output format** | The structure of the final deliverable — charts, tables, narrative summary |
| **Schema binding** | Tied to a specific schema version. If the schema changes, Actioneer auto-adapts or flags the mismatch |

---

## Creating a Playbook

| Step | Description |
| :--- | :--- |
| **Run the analysis** | Ask your question in Chat, refine the results, confirm the methodology is correct |
| **Save as Playbook** | Actioneer extracts the steps automatically — queries, intermediate logic, output format, and insight generation |
| **Review and adjust** | Edit parameters, adjust branching logic, or add validation checks |

---

## Running a Playbook

| Trigger | Description |
| :--- | :--- |
| **On demand** | Run manually from the Playbooks page or reference it in Chat |
| **On schedule** | Set a cadence (daily, weekly, every six hours) and Actioneer runs it automatically |
| **Via Scouts** | Automated agents that run Playbooks in the background, grade results, and flag anomalies |

<Info>
  Scouts are Actioneer's scheduled monitoring agents. They execute Playbooks on a recurring basis and deliver graded reports — A through E — per title, region, or team. Anomalies are surfaced with root-cause context attached.
</Info>

---

## Schema Drift Handling

When your data warehouse schema changes — a column is renamed, a table is restructured — Playbooks do not silently break. Actioneer detects the mismatch, attempts to auto-adapt the affected queries, and creates a new versioned copy. If auto-adaptation fails, the Playbook is flagged for manual review.

---

## Playbook Discovery

Before creating a new Playbook, Actioneer checks whether a similar one already exists using semantic matching at an 80%+ similarity threshold. This prevents duplication and keeps the Playbook library clean.
