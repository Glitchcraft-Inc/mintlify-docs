# Playbooks

The analysis your team runs every week — **automated, versioned, on autopilot.**

A Playbook is a saved, reusable, multi-step analysis pipeline — a sequence of data queries, computations, and visualisations that can be run repeatedly with consistent results. UA reviews, churn scans, lifecycle analyses — built once, run automatically. Weeks of analyst effort, on autopilot.

---

## Creating a Playbook

### From a conversation

Ask Actioneer to create one:
- *"Create a playbook for weekly retention analysis"*
- *"Build a playbook that tracks CPI by campaign monthly"*

Actioneer redirects you to the Playbook Builder, where a guided process walks you through defining each step.

### Guided Q&A Playbook

The Playbook Builder runs a structured interview:
- What metrics do you want to track?
- What date ranges and filters apply?
- What breakdowns (country, platform, channel) do you need?
- What output format — tables, charts, or both?

Based on your answers, it auto-generates the full playbook with validated queries.

### AI-assisted generation

Describe your goal in plain language and Actioneer will break it into logical steps, write the SQL and code, add visualisations, and validate against your schema.

*"Create a playbook that pulls D1, D7, D30 retention by install cohort, computes drop-off rates, and produces a trend chart"*

## Playbook Structure

| Component | Description |
| :--- | :--- |
| **Trigger conditions** | What initiates the Playbook — a schedule, an anomaly, or a manual run |
| **Analytical steps** | Ordered sequence of SQL queries, intermediate analysis, and validation checks |
| **Decision logic** | Conditional branching based on intermediate results |
| **Action steps** | Push a Segment, send an alert, update a Board, or trigger a follow-up Playbook |
| **Approval gates** | Optional human-in-the-loop checkpoints before critical actions execute |
| **Outcome logging** | Every run is logged — inputs, outputs, decisions, and execution time |

## Playbook Features

### Chat with your Playbook

Once a Playbook exists, you can have a conversation about it:
- *"Run my retention playbook for April 2026"*
- *"What did the last run of my revenue playbook show?"*
- *"Explain the results from cell 3"*

Actioneer loads the Playbook, executes it, and walks you through the results.

### Add to a Board

Playbook outputs feed directly into Boards:
- *"Add my retention playbook results to the weekly performance board"*
- *"Create a new board from my monetisation playbook"*

This connects live Playbook runs to dashboards for ongoing monitoring.

## Managing Playbooks

| Action | How to do it | Example prompt |
| :--- | :--- | :--- |
| **Run** | Execute on demand or reference in a conversation | *"Run my retention playbook"* |
| **Schedule** | Set a cadence — daily, weekly, every six hours | Configured in Playbook settings |
| **Edit** | Modify steps, queries, filters, or visualisations | *"Edit my retention playbook to add D14 retention"* |
| **Delete** | Permanently remove — confirmation required | *"Delete the weekly retention playbook"* |

---

## Schema Drift Handling

When your warehouse schema changes, Playbooks do not silently break. Actioneer detects the mismatch, attempts to auto-adapt the affected queries, and creates a new versioned copy. If auto-adaptation fails, the Playbook is flagged for review.

---

## Playbook Discovery

Before creating a new Playbook, Actioneer checks whether a similar one already exists using semantic matching — preventing duplication.
