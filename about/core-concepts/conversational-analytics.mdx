# Conversational Analytics

Plain-language access to the data the business runs on — grounded in verified definitions, not model assumptions. Conversational analytics replaces the queue of "can someone in data pull this for me," gives Growth, Finance, and Ops the ability to answer their own questions in seconds, and reconciles the multiple-answers-to-the-same-question problem because everyone asks against the same context layer.

---

## How It Works

| Step | Description |
| :--- | :--- |
| **Ask a question** | Write in plain English. No SQL, no query syntax, no specific formatting. |
| **Context loaded** | The Context Engine loads your metric definitions, north-star goals, and business relationships. |
| **Agents dispatched** | One question can trigger agents across every corner of the data estate simultaneously. |
| **SQL generated** | Queries are generated, validated against governed definitions, and executed against your warehouse. |
| **Answer delivered** | Results returned as a chart, table, or metric — with the SQL and reasoning visible. |

---

## What You Can Ask

| Question Type | Example |
| :--- | :--- |
| **Metric lookups** | "What was DAU last week?" |
| **Breakdowns** | "Show me retention by acquisition channel for the March cohort." |
| **Comparisons** | "Compare Android vs iOS funnel conversion for Q1." |
| **Anomaly investigation** | "Why did signups drop on Tuesday?" |
| **Segment exploration** | "How many users completed KYC but never invested?" |
| **Time-series** | "Plot weekly revenue for the last 12 weeks." |
| **Flow paths (Sankey)** | "Show me the user flow from install to first purchase — where do they go after onboarding?" |

---

## Why It's Different

| Generic AI query tool | Actioneer Conversational Analytics |
| :--- | :--- |
| Model reasons over raw data | Model reasons over the Context Engine — governed, business-specific |
| Answers may hallucinate | Low-confidence queries flagged — no hallucinations permitted |
| Starts from scratch every conversation | Context compounds — every interaction refines the platform |
| One query, one table | One question, full coverage — agents span the entire data estate |

<Info>
  Every answer is traceable. The SQL query, data source, and execution time are displayed alongside the result. Every team operates on the same context — every answer carries reasoning.
</Info>

<Tip>
  Full parity with Mixpanel-style report shapes — funnels, retention cohorts, flow paths, segment breakdowns — is achievable and has been delivered for customers replacing Mixpanel directly.
</Tip>

## From Answer to Action

Any result can flow into other parts of the platform.

| Action | Description |
| :--- | :--- |
| **Pin to a Board** | Save a chart or metric to a persistent dashboard |
| **Save as a Playbook** | Codify a multi-step analysis so it runs automatically |
| **Build a Segment** | Turn a cohort into a targetable audience and push to CLM |
| **Set an alert** | Monitor a metric and get notified when it crosses a threshold |
