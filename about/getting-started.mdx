# Getting Started

Two weeks to a working platform. Four or eight weeks to the first operational capability live in production. Actioneer works alongside the owner of each data system to confirm definitions and access, then handles all engineering — no engineering is required from your team.

---

## Phase 1 — Discovery

Data systems surveyed. Workflows audited. First capability scoped.

| Step | Description |
| :--- | :--- |
| **Data audit** | Actioneer maps your connected data systems — warehouses, BI tools, CDPs, CRMs, ad platforms |
| **Workflow review** | Existing analytical workflows are audited — weekly reviews, funnel checks, cohort analyses, reporting cadences |
| **Capability scoping** | The first production capability is identified — typically the highest-ROI, most-requested workflow |
| **Access provisioning** | Connect data sources via the integration guides. Actioneer operates read-only by default. |

## Phase 2 — Foundation

Definitions, ownership, and relationships codified into the Context Engine.

| Step | Description |
| :--- | :--- |
| **Metric definitions** | Define what "conversion," "retention," and your north-star metrics mean at your company. Codified once, used across every capability. |
| **Ownership graph** | Map which team or person owns each data asset — tables, dashboards, playbooks |
| **Business context** | KPI targets, campaign history, event taxonomy, and causal relationships loaded into the Context Engine |
| **Validation** | Your data team governs definitions. Actioneer builds on top. |

<Tip>
  During onboarding, Actioneer's team runs a discovery process to map existing definitions from BI tools, spreadsheets, and documentation. The platform starts with accurate, team-validated context from day one.
</Tip>

## Phase 3 — Pilot and Production

First capability ships. Operational refinement. Adoption. Scope for next capability begins.

### 4-week pilot (standard)

| Week | Focus |
| :--- | :--- |
| **1** | Connect data systems. Codify the first slice of the context layer. Chat interface live on real data. |
| **2** | Refine context. Identify and scope the first operational capability. |
| **3** | Build the first capability. Test against real data. |
| **4** | Deploy to production. Refine. Scope next capability. |

### 8-week pilot (larger scope)

| Weeks | Focus |
| :--- | :--- |
| **1–2** | Connect data systems across a wider footprint. Full context layer in place. Chat live. |
| **3–4** | First capability in production. Multi-team rollout planning. |
| **5–6** | Second capability built. First refined. |
| **7–8** | Third capability or deeper integration. Operational handoff. Engagement transitions to ongoing. |

### Ongoing

| Step | Description |
| :--- | :--- |
| **Operational refinement** | Playbooks are tuned, Scouts are calibrated, alert thresholds are adjusted based on real usage |
| **Expansion** | New capabilities are built as the business identifies them. The platform compounds — each new capability builds on the context already established. Most engagements grow over time. |

---

## What Engineering Is Required?

None for the build. Actioneer works alongside the owner of each data system to confirm definitions and access, then handles all engineering. Your team's responsibility is operational sign-off and adoption.

## Common Questions

    Two weeks to a working platform. Four weeks (standard) or eight weeks (larger scope) to the first operational capability live in production. In-house builds take 6–12 months.

  <Accordion title="How many data sources does Actioneer connect to?">

    No. Actioneer connects to data in place — read-only by default. Nothing is ingested, copied, or relocated. Write access is scoped per capability where required. For institutions that require it, Actioneer runs entirely on open-source models inside the customer's perimeter.

  <Accordion title="Can Actioneer run inside our own infrastructure?">
    Yes. Full platform deployable inside your own infrastructure. Open-source models can run entirely inside your perimeter. Private cloud deployment available.
  </Accordion>
  <Accordion title="What certifications does Actioneer hold?">
    SOC 2 Type II, ISO 27001, GDPR, HIPAA-aligned (BAA available), DPDP Act (India) compliant. Private cloud deployment and data residency options (US, EU, APAC) are available.
