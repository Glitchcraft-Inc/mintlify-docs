# Getting Started

Actioneer engagements follow three phases — from discovery to production. No engineering is required from your team for the build. Actioneer works alongside the owner of each data system to confirm definitions and access, then handles all engineering.

---

## Phase 1 — Discovery

Data systems surveyed. Workflows audited. First capability scoped.

| Step | Description |
| :--- | :--- |
| **Data audit** | Actioneer maps your connected data systems — warehouses, BI tools, CDPs, CRMs, ad platforms |
| **Workflow review** | Existing analytical workflows are audited — weekly reviews, funnel checks, cohort analyses, reporting cadences |
| **Capability scoping** | The first production capability is identified — typically the highest-ROI, most-requested workflow |
| **Access provisioning** | Connect data sources via the integration guides. Actioneer operates read-only by default. |

## Phase 2 — Foundation

Definitions, ownership, and relationships codified into the Context Engine.

| Step | Description |
| :--- | :--- |
| **Metric definitions** | Define what "conversion," "retention," and your north-star metrics mean at your company. Codified once, used across every capability. |
| **Ownership graph** | Map which team or person owns each data asset — tables, dashboards, playbooks |
| **Business context** | KPI targets, campaign history, event taxonomy, and causal relationships loaded into the Context Engine |
| **Validation** | Your data team governs definitions. Actioneer builds on top. |

<Tip>
  During onboarding, Actioneer's team runs a discovery process to map existing definitions from BI tools, spreadsheets, and documentation. The platform starts with accurate, team-validated context from day one.
</Tip>

---

## Phase 3 — Production

First capability ships. Operational refinement. Adoption. Scope for next capability begins.

| Step | Description |
| :--- | :--- |
| **First capability live** | The scoped capability ships to production — a Playbook, a Scout, a conversational analytics workflow, or a custom agent |
| **Team onboarding** | Users start asking questions. Every interaction refines the Context Engine. |
| **Operational refinement** | Playbooks are tuned, Scouts are calibrated, alert thresholds are adjusted based on real usage |
| **Expansion** | Scope for the next capability begins. The platform compounds — each new capability builds on the context already established. |

---

## What Engineering Is Required?

None for the build. Actioneer works alongside the owner of each data system to confirm definitions and access, then handles all engineering. Your team's responsibility is operational sign-off and adoption.

## Common Questions

<AccordionGroup>
  <Accordion title="How long does it take to get the first capability live?">
    Weeks, not months. The first engagement establishes the platform and ships the first production capability. Enterprise competitors or in-house builds take 6–12 months.
  </Accordion>
  <Accordion title="How many data sources does Actioneer connect to?">
    700+ out of the box. A custom API is available for sources not natively supported.
  </Accordion>
  <Accordion title="Does our data leave our environment?">
    No. Actioneer connects to data in place — read-only by default. Nothing is ingested, copied, or relocated. Write access is scoped per capability where required.
  </Accordion>
  <Accordion title="Can Actioneer run inside our own infrastructure?">
    Yes. Custom private cloud deployment is available for institutions that require it.
  </Accordion>
  <Accordion title="What certifications does Actioneer hold?">
    SOC 2 Type II, ISO 27001, GDPR. Private cloud deployment and data residency options (US, EU, APAC) are available.
  </Accordion>
</AccordionGroup>
