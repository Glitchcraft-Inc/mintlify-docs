# Core Concepts

Actioneer's capabilities fall into four patterns — each built on top of the Context Engine and Agent Orchestration Runtime. Alongside these, the platform includes purpose-built features for ongoing analysis and monitoring.

---

## Capability Patterns

| Pattern | What it does | Key outcome |
| :--- | :--- | :--- |
| **Conversational Analytics** | Plain-language access to data, grounded in governed definitions | 3-day analyst requests answered in the same conversation |
| **Monitoring Agents (Scouts)** | Scheduled agents that proactively surface what's moving in the business | Anomalies caught before they compound, with root cause attached |
| **Operational Playbooks** | Multi-step workflows combining analysis, decision, and action | Weeks of analyst effort, on autopilot |
| **Custom Agents** | Bespoke agentic capabilities built to the shape of your organisation | Variance commentary, reconciliation, exception handling — governed and observable |

---

## Platform Features

| Feature | Type | Purpose |
| :--- | :--- | :--- |
| **Boards** | Visualisation | Persistent pages of charts, tables, and metrics for ongoing monitoring |
| **Segments** | Audience + Action | Rule-based user groups — defined by behaviour, pushed to CLM tools |
| **Funnels** | Analysis | Step-by-step conversion tracking with drop-off diagnosis |
| **Retention** | Analysis | Cohort-based retention tracking across time windows and segments |
| **Metrics** | Data Catalog | Governed metric definitions, SQL logic, and the data catalog |
| **Knowledge** | Context | Institutional knowledge that sharpens every AI answer |

---

## How They Connect

```mermaid
flowchart LR
    A["Ask\nConversational Analytics"] --> D["Detect\nScouts & Alerts"]
    D --> An["Analyse\nFunnels · Retention · Boards"]
    An --> Act["Act\nSegments · Playbooks · Campaigns"]
    Act --> L["Learn\nContext Engine compounds"]
    L --> A
```

<Tip>Every interaction feeds back into the Context Engine. The platform gets sharper with use.</Tip>

## Use Cases by Buyer Function

<Tabs>
  <Tab title="Growth" icon="trending-up">
    | Problem | Capability Pattern |
    | :--- | :--- |
    | Can't answer growth questions same-day — data scattered across product / ads / CRM / messaging | Conversational Analytics |
    | Don't know which cohorts to target with which messaging | Conversational Analytics + Scouts |
    | Funnel performance is opaque — can't isolate where users drop and why | Conversational Analytics (Funnel parity) |
    | Retention diagnosis requires the data team and takes weeks | Custom Agent: retention diagnostic |
    | Performance marketing spend partially wasted but the team can't say where | Scouts + Custom analysis |
    | Cross-sell opportunities exist but the messaging stack doesn't reach them | Playbooks (triggered messaging) |
  </Tab>
  <Tab title="Retention" icon="refresh-cw">
    | Problem | Capability Pattern |
    | :--- | :--- |
    | Don't know who's about to churn or why | Custom Agent: churn diagnostic + Scout |
    | Lifecycle has too many touchpoints to instrument manually | Playbooks across the lifecycle |
    | KYC / activation drop-off is invisible | Scout surfacing the funnel drop and reasoning |
    | Re-engagement campaigns are generic when they should be cohort-specific | Playbooks (cohort-specific messaging) |
  </Tab>
  <Tab title="Operations" icon="settings">
    | Problem | Capability Pattern |
    | :--- | :--- |
    | Supplier/vendor risk visible too late, after damage done | Scout ranked by customer impact |
    | Exception handling eats the day | Playbook + Custom Agent for triage |
    | Production schedule slippage is reactive, not proactive | Scout + Playbook |
    | Cross-system reconciliation (ERP / CRM / billing) is manual | Custom Agent: continuous reconciliation |
  </Tab>
  <Tab title="Finance / FP&A" icon="calculator">
    | Problem | Capability Pattern |
    | :--- | :--- |
    | Variance commentary eats the team every period | Custom Agent: variance commentary + drivers |
    | Board-pack prep is a fire drill | Custom Agent: board variance pack |
    | Period close requires cross-system reconciliation done by hand | Custom Agent: period close assistant |
    | Forecast accuracy is unknown until it's wrong | Scout: forecast accuracy diagnostic |
  </Tab>
</Tabs>
