# Actioneer

Actioneer is an AI coworker for growth and business teams. It sits on top of your existing data stack — warehouses, BI tools, CDPs, and CLMs — and lets anyone on the team go from question to insight to action, all in natural language, without switching tools.

---

## Key Components

Actioneer is built around three layers that work together on top of your existing infrastructure.

| Component | Purpose |
| :--- | :--- |
| **Query Layer** | A natural-language interface to your data warehouse. Ask a question in plain English, get an answer grounded in your schema and metric definitions. |
| **Memory Layer** | Persistent business context — KPI definitions, past campaigns, playbooks, and team preferences. Every conversation builds on the last. |
| **Action Layer** | Act on insights from the same window. Build a segment, push a campaign, set an alert, or deploy a monitoring agent — without switching tools. |

The combination is the point. A BI tool gives you the Query Layer. A CDP gives you fragments of the Action Layer. Actioneer ties all three together with compounding business context in between.

---

## Who Is Actioneer For

Actioneer is built for operators at B2C and fintech companies — people who need answers from data every day but do not want to wait on analyst queues or learn SQL.

| Role | How they use Actioneer |
| :--- | :--- |
| **Growth Lead** | Ask ad-hoc questions in plain English, get granular breakdowns in seconds |
| **Product Manager** | Monitor funnel health, spot anomalies early, act on them immediately |
| **Marketing Team** | Build hyper-targeted segments and push campaigns from the same window |
| **Business Analyst** | Offload tier-one requests, focus on deeper modelling and experimentation |

---

## Where Actioneer Sits in the Stack

Actioneer is not a replacement for your data warehouse, BI tool, or CDP. It is an intelligence layer that sits across all of them.

| Stack Layer | Tools | Role |
| :--- | :--- | :--- |
| **Raw Data** | App events, backend databases, transaction logs | Source of truth |
| **Data Warehouse** | Snowflake, BigQuery, Databricks, Redshift, S3 | Stores and structures data for analytical queries |
| **BI & Analytics** | Amplitude, Mixpanel, Power BI, Looker, Tableau | Predefined dashboards and visualisation |
| **CDP / CLM** | CleverTap, MoEngage, WebEngage, Braze | Customer profiles and lifecycle messaging |
| **Actioneer** | Query + Memory + Action | Natural-language querying, persistent context, and insight-to-action — across all layers above |

<Info>
  Actioneer connects to your existing infrastructure. It does not require data migration, pipeline rebuilds, or schema changes.
</Info>

## Actioneer vs Alternatives

| Alternative | What they do well | Where Actioneer differs |
| :--- | :--- | :--- |
| **BI Tools** (Tableau, Looker) | Predefined dashboards, visualisation | Natural-language ad-hoc queries, action layer, memory |

| **Databricks Genie** | Text-to-SQL on Databricks warehouse | Business context memory, action orchestration across tools |
| **Claude + MCP** | General LLM with data access | Domain-specific, remembers KPIs and playbooks, integrated action |
| **Internal Analytics Team** | Deep custom analysis | 100x faster turnaround, self-serve for non-analysts |

## Security and Compliance

Actioneer is SOC 2 Type II certified, ISO 27001 certified (audited by Schellman), and GDPR compliant. All data is encrypted with AES-256 at rest and TLS 1.3 in transit. Role-based access control, audit logs, and explainable AI are built in.
